= Course Summary

Congratulations on completing the Aura Agent course!

You have designed, built, and published a Neo4j Aura Agent that answers natural language questions using your knowledge graph — and connected it to an AI application through MCP.

== Understanding Aura Agent

You now know how to:

* Understand what Aura Agent is and when to use it
* Explain how agents use multi-step reasoning to interpret questions, select tools, and generate responses
* Distinguish between the three tool types: Cypher Template, Text2Cypher, and Similarity Search
* Enable Aura Agent in your organization settings and manage project-level access

== Building an Agent

You now know how to:

* Create an agent with AI from a prompt and your instance schema
* Add Cypher Template tools with named, typed parameters and instructive descriptions
* Add Text2Cypher tools for ad-hoc questions where the query structure changes at runtime
* Design an agent with a clear role, scope, and question-to-tool mapping
* Edit, delete, and manage tools and agent configuration

== Publishing and Connecting

You now know how to:

* Choose between Internal and External access modes
* Enable the MCP server and expose your agent as a callable tool for AI applications
* Connect your agent to Cursor using the MCP endpoint and authenticate through Aura
* Test your agent in Cursor and verify tool selection in the reasoning panel

== Best practices

=== Design

* Keep agents focused on one domain. A narrow scope makes tool selection easier for the LLM and produces more predictable results.
* Use Cypher Templates for any question where you can write the full Cypher now with only `$parameter` slots for variable values. Reserve Text2Cypher for questions whose structure changes.
* Give each distinct question pattern its own tool. A tool covering unrelated questions will have a vague description and get picked at the wrong time.

=== Writing Good Descriptions

* Write tool descriptions that answer two questions: what does this tool return, and when should it be used instead of the others?
* Write parameter descriptions as instructions with examples — "The customer ID, for example ALFKI or QUICK" — not just labels.
* For Text2Cypher, state when to use it ("only when no other tool fits") and include domain-specific context such as key node labels and relationship types.

=== Testing and Maintenance

* Use the reasoning panel to verify which tool the agent selected and what Cypher was generated, especially for Text2Cypher.
* Test Text2Cypher output before relying on it in production. The generated Cypher can vary between calls.
* After adding, editing, or deleting tools, always click **Update agent** to persist the configuration.
