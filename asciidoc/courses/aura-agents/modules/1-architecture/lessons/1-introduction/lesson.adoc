== Introduction

Aura Agents let you build agents that query your knowledge graph from the Aura console without writing code. Understanding what they are and when to use them helps you choose the correct approach for your application.

Here is a short example of an agent answering a question over a Northwind knowledge graph:

video::https://cdn.graphacademy.neo4j.com/courses/aura-agents/agent-demo.mp4["Aura Agent answering a question over Northwind", role="cdn", width=100%]

In this lesson, you will learn what Aura Agents are, when to use them, the agent lifecycle, and how they fit into the Neo4j ecosystem.

== Prerequisites and pricing

To create an Aura Agent, you need:

* Have a knowledge graph in a **running AuraDB instance**
* Enable **Generative AI assistance** in your Organization settings
* Toggle on **Aura Agent**, available when Generative AI assistance is enabled

[WARNING]
.Keep Generative AI assistance enabled
====
Disabling Generative AI assistance automatically disables Aura Agent.
====

=== Visibility

Your agent's visibility controls who can access it.

When you first create your agent, its visibility is set to **Internal**. You can add tools and test your agent in the Aura Console. While the agent is internal, it is only accessible to members of your Aura project, and it is free.

Setting the visibility to **External** exposes your agent through a Model Context Protocol (MCP) server endpoint, so you can connect it to a host such as Claude Desktop or Cursor. Setting an agent to external incurs charges — see link:https://neo4j.com/pricing/[Neo4j pricing^] for details.

== The Aura Agent lifecycle

An Aura Agent does not answer in one step. It follows five steps:

. **Receive**: User query or message
. **Reason**: LLM plans which tools to use and in what order
. **Act**: Invoke tools: Cypher Template, Similarity Search, or Text2Cypher
. **Observe**: Incorporate tool results into context
. **Respond**: Generate final answer or take next action

The diagram below illustrates this workflow: from user input through reasoning, tool selection and execution, to final output.

image::images/agent-process.svg[Agent workflow: user input, reasoning, tool selection, tool execution, and final output]

The agent must understand user intent before acting. The LLM selects the correct tool for the query. Tools execute and return structured results. The agent adds tool output to its context for the next reasoning step, then synthesizes context into a natural language answer.


== Trace through a query

Consider an agent with three tools: a "Get Customer" Cypher Template, a "Top Customers by Order Count" Cypher Template, and Text2Cypher. A user asks: "Who has ordered Pavlova repeatedly?"

The first two tools return customers by ID or a ranked list. Neither finds repeat Pavlova buyers. The agent selects Text2Cypher to generate and run a query that answers the question.

. **Receive**: The agent receives the question
. **Reason**: The LLM identifies that finding repeat Pavlova buyers requires order history and customer details, so it chooses Text2Cypher

image::images/reasoning-pavlova-repeatedly-text2cypher.png[Agent reasoning panel showing thought process and Text2Cypher selection]
. **Act**: The agent invokes Text2Cypher, which generates Cypher and executes it

. **Observe**: The agent adds customer names and order counts to context
. **Respond**: The LLM formats a natural language answer



== When to use Aura Agents

Aura Agents are useful when:

* You want to expose knowledge graph data using natural language, without writing application code
* You need to integrate graph-backed answers into Claude Desktop or Cursor through MCP

Aura Agent tools query only your graph. To combine graph data with external APIs, build a separate AI application or multi-agent system.
