== Introduction

Aura Agents let you build agents that query your knowledge graph from the Aura console without writing code.

In this lesson, you will learn:

* What Aura Agents are
* When to use Aura Agents
* How Aura Agents work

== What are Aura Agents?

A **Neo4j Aura Agent** is a no/low-code platform in the Aura Console where reasoning and graph queries run inside Aura's infrastructure. You connect the agent to an AuraDB instance, give a prompt, and the agent handles the rest.

Aura Agents are useful when you want to expose knowledge graph data using natural language, without writing application code

== How Aura Agents Work

Aura Agents use reasoning to break down each request into a sequence of steps.

This is often called chain-of-thought or multi-step reasoning: the model doesn’t jump straight to an answer, it reasons step by step and can use tools in between.

In the context of Aura Agents, the steps are:

* **Interpret user input**: Obtain the request, decide what information is needed, and determine which tool is most likely to answer.
* **Execute tools**: Run the selected tools, retrieve results, and decide whether to run more tools or generate a final answer.
* **Generate response**: Generate a natural language response based on the retrieved data.

== Tool Types

Aura Agents use three read-only tool types to query your graph:

* **Cypher Template**: Runs predefined Cypher queries with parameters you define.
* **Text2Cypher**: Generates Cypher from natural language at runtime using your schema.
* **Similarity Search**: Finds nodes by similarity using vector embeddings.

Vectors are numeric representations of data that can be used when you want to compare or search for similarities.

Embeddings are those vectors produced by a model so that similar things end up close together.

If your data contains vector embeddings, mark the "This data has vector embeddings" checkbox, and the agent will be able to create a Similarity Search tool to find nodes that are similar to the user's question.


The user's question is converted to a vector and compared against embeddings stored as node properties, so the agent can answer "what is like this?" without exact keyword matches. This is useful when you want to find nodes that are similar to the user's question, but you don't know the exact keywords to use.

== Quick Start

The **Create with AI** option generates a working agent from a given prompt and your instance schema.

Your prompt and schema drive what gets created. The agent maps your described use case to node labels, relationship types, and properties in the schema to build tools that query the right parts of your graph. 

In the end you will have a working agent that can answer questions about your data.

video::https://cdn.graphacademy.neo4j.com/courses/ai-agents/create-with-ai.mp4["Create with AI", role="cdn", width=100%]
