== Components

An Aura Agent consists of five components:

* **LLM**: Language model that reasons and generates responses
* **Tools**: Cypher procedures, APIs, or custom functions the agent can call
* **Knowledge base**: Neo4j graph for structured data and graph-native retrieval
* **Memory**: Conversation history and persistent context
* **Orchestrator**: Coordinates reasoning, tool calls, and response generation

[.slide]
== Role of Each Component

* **LLM**: Interprets user intent, selects tools, and generates natural language answers
* **Tools**: Extend the agent beyond text. Cypher tools query the graph; API tools call external services.
* **Knowledge base**: The graph stores facts. Cypher and GraphRAG retrieve them.
* **Memory**: Enables multi-turn conversations and persistent context
* **Orchestrator**: Manages the workflow from user input through LLM reasoning, tool invocation, result integration, and response


[.slide]
== How Components Work Together

Example flow:

. User asks a question → **Orchestrator** passes it to the **LLM**
. **LLM** reasons that the "Get Customer" tool is needed
. **Orchestrator** invokes the **Tool**, which runs Cypher against the **Knowledge base**
. **Tool** returns rows → **Orchestrator** adds them to context
. **LLM** generates a response using the tool results
