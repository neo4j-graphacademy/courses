[.question]
= Reading the Reasoning Panel

What does the **Applying agent tool** section in the agent's reasoning panel show you?

* [ ] The LLM's confidence score for the answer it generated
* [x] The tool selected, the parameter values extracted from the question, and the raw output returned
* [ ] The total execution cost of the Cypher query in milliseconds
* [ ] A list of all tools the agent considered before making a selection

[TIP,role=hint]
.Hint
====
The "Applying agent tool" section appears after the LLM has already chosen a tool. What information would help you verify the tool ran correctly?
====

[TIP,role=solution]
.Solution
====
**The tool selected, the parameter values extracted, and the raw output.**

This section lets you verify three things: the agent picked the correct tool, the LLM extracted the correct values from the user's question (for example, `$customer_id = "ALFKI"`), and the tool returned useful results. For Text2Cypher tools, you can also read the generated Cypher directly and check it against your schema.

**Why the others are wrong:**

* Confidence score: Aura Agents do not expose a numeric confidence score.
* Execution cost: Query execution time is not shown in the reasoning panel.
* All tools considered: The Reasoning section (not Applying agent tool) shows the LLM's thought process about which tool to use.

**Recall:** The Trace through the query section in this lesson walks through both the Reasoning and Applying agent tool sections.
====
