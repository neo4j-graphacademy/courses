= Agent Project Permissions

A teammate can open your agent and ask questions in the preview, but they do not see the option to configure the agent or change its tools. Why?

Select the correct answer.

* [ ] The agent is set to Internal. Only External agents can be configured by others
* [ ] They need to enable the MCP server first
* [x] Only Project Admins can create, update, or delete agents. They likely have Viewer or Member role
* [ ] They must be added to the agent's allow list

[TIP,role=hint]
.Hint
====
Project roles control who can change agent configuration. Listing and invoking agents is allowed for more roles than creating or editing agents.
====

[TIP,role=solution]
.Solution
====
**Only Project Admins can create, update, or delete agents.**

Viewers and Members can list and invoke agents. They cannot change configuration, tools, or access settings. If your teammate does not see Configure or cannot edit the agent, they have Viewer or Member role. An Organization or Project Admin must grant them Project Admin role if they need to manage agents.
====
