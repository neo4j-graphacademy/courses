= Internal vs External Access

What is the difference between Internal and External access for an Aura Agent?

* [ ] Internal agents support fewer tool types than External agents
* [ ] Internal agents can only use Cypher Template; External agents support all tool types
* [ ] Internal agents are free and faster; External agents have higher latency
* [x] Internal agents are only accessible within the Aura Console; External agents are accessible through REST API and MCP server endpoints by external applications

[TIP,role=hint]
.Hint
====
Think about who or what can send messages to the agent in each mode.
====

[TIP,role=solution]
.Solution
====
**Internal agents are only accessible within the Aura Console; External agents are accessible through REST API and MCP server endpoints.**

Internal is the default mode for development and testing. Only project members using the Aura Console preview panel can interact with the agent. External exposes the agent through endpoints that external applications (such as Cursor or Claude Desktop) can connect to through MCP.

**Why the others are wrong:**

* Fewer tool types: Both modes support the same tool types (Cypher Template, Text2Cypher, Similarity Search).
* Only Cypher Template for Internal: Tool type availability is not affected by the access mode.
* Faster Internal: Latency is not determined by the access mode.

**Recall:** The Internal and External Access section in this lesson describes both modes and their use cases.
====
