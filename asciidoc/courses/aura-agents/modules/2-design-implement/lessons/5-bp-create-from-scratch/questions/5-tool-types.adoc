[.question]
= Limits of Agent Tools

A stakeholder wants the agent to call an external REST API when the graph does not contain the answer. What can you tell them about Aura Agent tools?

Select the correct answer.

* [ ] Add a Text2Cypher tool that generates the API request from the user's question
* [ ] Add a custom tool in the agent that calls the REST API when the graph has no answer
* [x] Agent tools only read from your graph. Calling external REST APIs would require a different integration, such as an app that uses the agent and the API
* [ ] Enable External access so the agent can reach external endpoints

[TIP,role=hint]
.Hint
====
Cypher Template and Text2Cypher tools are read-only and run against your graph. What does that imply for external APIs?
====

[TIP,role=solution]
.Solution
====
**Agent tools only read from your graph. External APIs need a different integration.**

Cypher Template and Text2Cypher query your graph only. They do not call REST APIs or other services. To combine graph answers with external data, you would build an application that calls the agent over MCP or REST and also calls the API, then merges the results. External access only exposes the agent to external clients. It does not let the agent call out to other APIs.
====
