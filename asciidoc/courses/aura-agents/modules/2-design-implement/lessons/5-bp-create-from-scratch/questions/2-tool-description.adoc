[.question]
= Text2Cypher Tool Description

Why should a Text2Cypher tool description include node labels and relationship types?

Select the correct answer.

* [ ] They are required fields in the Aura tool form
* [ ] They are shown in the agent's reply to the user
* [x] The LLM has no other schema access. Without them it may generate Cypher with wrong relationship names
* [ ] They are used by Neo4j to optimize the query plan

[TIP,role=hint]
.Hint
====
Text2Cypher generates Cypher at runtime. The model only sees what you put in the description. What would happen if relationship types were missing?
====

[TIP,role=solution]
.Solution
====
**The LLM has no other schema access. Without labels and relationship types it may generate invalid Cypher.**

The LLM sees only the tool description when generating Cypher. If you do not list node labels and relationship types, it will guess and can use names that do not exist in your graph. The description is not a required form field, is not shown to the user, and does not affect Neo4j's query planning.
====
