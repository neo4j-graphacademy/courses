= Persisting Configuration Changes

You add a new Cypher Template, run a test in the preview, and the agent still does not use the new tool. What step is missing?

Select the correct answer.

* [ ] Run a question that explicitly names the new tool so the LLM can match it
* [x] Click **Update agent** to save the configuration. Edits are not applied until you persist them
* [ ] Reconnect the agent to the instance so it picks up the new tools
* [ ] Refresh the browser to reload the tool list

[TIP,role=hint]
.Hint
====
The lesson says: after any change to tools—add, edit, or delete—click Update agent to save. The agent keeps using the previous configuration until you do. What step is the user missing?
====

[TIP,role=solution]
.Solution
====
**Click Update agent to save the configuration.**

Adding, editing, or removing tools is applied only when you click **Update agent**. The preview uses the previously saved configuration until then. Reconnecting, refreshing, or rephrasing the test question does not persist changes.
====
