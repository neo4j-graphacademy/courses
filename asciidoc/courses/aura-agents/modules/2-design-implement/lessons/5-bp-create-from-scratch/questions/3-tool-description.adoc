[.question]
= What a Good Tool Description Includes

What information should a good tool description include to help the LLM choose the correct tool?

Select the correct answer.

* [ ] The Cypher query and parameter names
* [ ] Only the name of the node labels in the graph
* [x] What the tool returns and when it should be used instead of other tools
* [ ] The full database schema and system configuration

[TIP,role=hint]
.Hint
====
The lesson says the LLM selects tools from descriptions only and does not inspect the Cypher or parameters.
====

[TIP,role=solution]
.Solution
====
**What the tool returns and when it should be used instead of other tools.**

Descriptions should answer what the tool returns and when the LLM should use it instead of the others. The LLM does not read the Cypher body or parameter names to choose between tools. Listing only labels is too thin. Full schema and system configuration belong elsewhere. The Text2Cypher path gets schema automatically.
====
