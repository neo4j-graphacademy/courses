[.question]
= Template vs Text2Cypher

Which of these questions is best handled by Text2Cypher rather than a Cypher Template?

Select the correct answer.

* [ ] Top 10 customers by order count
* [ ] Get order details for order ID ALFKI-12345
* [ ] Top 5 products by revenue in the Electronics category
* [x] Which customers ordered products from more than two different suppliers?

[TIP,role=hint]
.Hint
====
Apply the test: can you write the full Cypher now with only `$parameter` slots for values? For the Text2Cypher case, the filter, the path, and what you aggregate depend on the question itself.
====

[TIP,role=solution]
.Solution
====
**Which customers ordered products from more than two different suppliers?**

For that question, the filter ("more than two"), the multi-hop path (Customer → Order → Product → Supplier), and the aggregation depend on the specific ask. You cannot pre-write one query and plug in parameters. The structure changes with the question, so Text2Cypher is appropriate.

The other three have fixed structure: same MATCH/WHERE/RETURN shape with only values (limit, ID, category name) changing. Those fit Cypher Templates.
====
