= Agent Scope Best Practice

Which approach gives the most predictable agent behavior?

* [ ] One agent that handles every use case across your organization
* [x] One main task per agent; create multiple agents for different capabilities
* [ ] Leave scope vague so the agent adapts to any question
* [ ] Add as many tools as possible to maximize coverage

[TIP,role=hint]
.Hint
====
The lesson recommends keeping the scope focused. One main task per agent works better than a do-everything agent.
====

[TIP,role=solution]
.Solution
====
**One main task per agent; create multiple agents for different capabilities.**

Keeping the scope focused reduces wrong tool selection and off-topic answers. If you need different capabilities, create multiple agents rather than one do-everything agent.
====

= Graph Schema in Prompts

Why include node labels and relationship types in your agent instructions?

* [ ] To allow the agent to modify the schema
* [ ] To reduce the number of tools the agent needs
* [x] To help the LLM construct valid Cypher when using Text2Cypher
* [ ] To improve response speed

[TIP,role=hint]
.Hint
====
The lesson states that mentioning node labels and relationship types helps with something related to query construction.
====

[TIP,role=solution]
.Solution
====
**To help the LLM construct valid Cypher when using Text2Cypher.**

Including the graph schema in your prompt gives the LLM the node labels and relationship types it needs to generate correct Cypher for ad-hoc questions.
====
