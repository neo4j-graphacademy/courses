= Cypher Template
:order: 1
:type: lesson

A **Cypher Template** is a pre-written, parameterized Cypher query. You define the query once and the LLM extracts parameter values from the user's question and runs it. Queries use named `$parameter` placeholders; the LLM fills them from the user's question at runtime.

Use a Cypher Template for common, repeated questions with known structure: "Get customer ALFKI", "List products in Beverages", "Top 10 customers by order count." Because the query is pre-written, results are deterministic: the same question always runs the same Cypher.

In the Aura Console, you add a Cypher Template from the agent configuration by clicking **Add Tool** → **Cypher Template**.

image::images/add-tool-menu.png[Add Tool menu showing Cypher Template, Text2Cypher, and Similarity Search options]

[TIP]
.Build templates first
====
Start with Cypher Templates for your core question types. Each template you add reduces how often the agent needs to generate Cypher dynamically.
====

== What is a parameter?

A **parameter** is a named placeholder in a Cypher query, prefixed with `$`, that gets its value supplied at runtime. Instead of hardcoding a value in the query, you write `$parameter_name` and the LLM extracts the correct value from the user's question and passes it in.

For example, this query has one parameter, `$customer_id`:

[source,cypher,role=noplay]
----
MATCH (c:Customer {id: $customer_id})
RETURN c.companyName, c.contactName, c.city
----

When a user asks "Get me customer ALFKI", the LLM extracts `"ALFKI"` from the question and runs the query with `$customer_id = "ALFKI"`. The same template handles "Get me customer QUICK" by substituting `"QUICK"`, with no new query required.

Parameters can hold different data types:

[cols="1,2,2"]
|===
|Type |Example parameter |Example question

|**string**
|`$customer_id`
|"Get customer ALFKI" → `customer_id = "ALFKI"`

|**string**
|`$category`
|"List products in Beverages" → `category = "Beverages"`

|**int**
|`$limit`
|"Show me the top 5 customers" → `limit = 5`
|===

Each parameter needs a name, a data type, and a description. The LLM uses the description to know what value to extract. Treat it as an instruction, not a label. A description like `"The customer ID"` is weaker than `"The Northwind customer ID, for example ALFKI or QUICK"`.

A Cypher Template tool in the agent configuration shows the name, description, parameters, and the Cypher query. Use the pencil icon to edit any of these fields.

image::images/cypher-template-tool-edit.png[Cypher Template tool edit dialog showing name, description, parameters, and Cypher query fields]

To add or edit a parameter, open the parameter dialog and set the name, data type, and description.
