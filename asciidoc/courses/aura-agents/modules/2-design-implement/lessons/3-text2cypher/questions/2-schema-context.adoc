= How the Description Affects Generated Cypher

The Text2Cypher tool automatically receives the database schema. How does the text you add in the tool description still affect the queries the model generates?

Select the correct answer.

* [ ] It has no effect because the schema alone determines all generated Cypher
* [ ] It only changes the agent's display name in the preview panel
* [x] It adds when-to-use, domain context, and guidance on entities and aggregations so the LLM can produce more accurate Cypher
* [ ] It forces the model to copy a fixed Cypher string you paste into the description

[TIP,role=hint]
.Hint
====
The schema is automatic. The description adds information the raw schema does not express: business rules, when to use this tool, and which fields matter for grouping or filters.
====

[TIP,role=solution]
.Solution
====
**It adds when-to-use, domain context, and guidance on entities and aggregations so the LLM can produce more accurate Cypher.**

The tool passes the question, the database schema, and the Text2Cypher system prompt to the model. Your description layers on when to use the tool, domain-specific aspects, relevant entities and categorical property patterns, and which attributes suit aggregation. That context steers generation beyond raw labels and relationship types alone.
====
