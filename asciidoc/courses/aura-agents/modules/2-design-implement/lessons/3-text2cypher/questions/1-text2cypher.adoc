= When a Template Is Not Enough

For "Which customers ordered products from more than two different suppliers?" why would you use Text2Cypher instead of a single Cypher Template with parameters?

Select the correct answer.

* [ ] Cypher Templates do not support aggregations or counts
* [x] The query pattern and filters depend on the question. You cannot pre-write one parameterized query that fits
* [ ] The graph might not have Customer–Order–Supplier relationships
* [ ] This question requires Similarity Search

[TIP,role=hint]
.Hint
====
With a template you fix the full query and only swap in values. Here, the filter ("more than two"), the path length, and what you aggregate depend on the question.
====

[TIP,role=solution]
.Solution
====
**The query pattern and filters depend on the question.**

The filter ("more than two"), the path (e.g. Customer → Order → Product → Supplier), and what you aggregate all depend on this specific question. You cannot write one query with `$parameter` slots that fits every similar question. Text2Cypher generates Cypher at runtime from the schema and the question. Cypher Templates do support aggregations when the pattern is fixed (e.g. "top N by count" with a `$n` parameter). Similarity Search is for vector similarity, not this kind of multi-hop query.
====
