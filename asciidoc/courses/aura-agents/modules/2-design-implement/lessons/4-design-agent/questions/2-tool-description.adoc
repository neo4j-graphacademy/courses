[.question]
= Writing Tool Descriptions

Why does a Text2Cypher tool description need to include node labels and relationship types?

* [ ] They are required fields in the Aura tool configuration form
* [ ] They appear in the agent's response to the user
* [x] The LLM uses them to generate valid Cypher. Without schema context it may produce wrong relationship types
* [ ] They speed up query execution by providing a query plan to the database

[TIP,role=hint]
.Hint
====
The LLM only sees the tool description when deciding what Cypher to generate. What does it need to avoid hallucinating relationship names?
====

[TIP,role=solution]
.Solution
====
**The LLM uses them to generate valid Cypher.**

Text2Cypher generates a Cypher query at runtime. The LLM has no access to your schema except through the description you write. Without node labels and relationship types, it will guess and may produce queries with relationship names that do not exist in your graph.

**Why the others are wrong:**

* Required fields: Node labels and relationship types are not enforced by the form. You write them as free text in the description.
* Appear in responses: Tool descriptions are internal instructions for the LLM, not visible to end users.
* Query plan: Neo4j generates its own query plan from the Cypher. The description has no effect on execution.

**Recall:** The tool descriptions section in this lesson shows a sample description that lists `Customer, Order, Product` nodes and their relationships explicitly.
====
