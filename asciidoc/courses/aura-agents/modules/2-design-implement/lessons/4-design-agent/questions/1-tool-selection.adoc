= Tool for an Aggregation Query

You need to answer "Show me the top 10 customers by order count". Which tool should you use?

* [x] Cypher Template: the `MATCH` pattern, aggregation, and `LIMIT` are fixed; only the limit value changes
* [ ] Text2Cypher: aggregation questions always require dynamic query generation
* [ ] Similarity Search: ranking by count is a semantic similarity problem
* [ ] Either tool works equally well for this question

[TIP,role=hint]
.Hint
====
Apply the test from the lesson: can you write the complete Cypher query right now, with only `$parameter` slots for variable values?
====

[TIP,role=solution]
.Solution
====
**Cypher Template is correct.**

The query structure is fully predictable: `MATCH`, `COUNT`, `ORDER BY`, `LIMIT $n`. Only the limit value changes depending on the user's question. Write it once as a template.

**Why the others are wrong:**

* Text2Cypher: Aggregations do not require dynamic generation when the structure is fixed. Text2Cypher is for questions where the `MATCH` pattern or `WHERE` conditions themselves change.
* Similarity Search: This tool finds semantically similar nodes using vector embeddings. It does not count or rank.
* Either: The design principle is Cypher Templates first. Reserve Text2Cypher for questions you genuinely cannot pre-define.

**Recall:** The question types table in this lesson defines the test: if the full query structure is fixed, use a Cypher Template.
====
