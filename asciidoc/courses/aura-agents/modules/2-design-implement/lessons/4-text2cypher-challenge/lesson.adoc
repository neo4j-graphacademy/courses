= Experimenting with Text2Cypher
:order: 4
:type: challenge

In this challenge, you will use your agent to trigger Text2Cypher, inspect the generated Cypher in the reasoning panel, and verify or improve the Text2Cypher tool description.

== Goal

Ask questions that only Text2Cypher can answer (no Cypher Template fits), then trace the generated Cypher and results. Optionally, improve the Text2Cypher tool description so the LLM generates correct relationship types and labels for your schema.

== Before you start

* Use the agent you created with AI in Module 1 (your knowledge graph or Northwind). That agent includes a Text2Cypher tool by default.
* Open the agent in the Aura Console (**Data Services** → **Agents** → select your agent).
* Ensure the preview panel is visible (the chat panel on the right where you can ask questions).

== Step 1: Ask a question that triggers Text2Cypher

In the preview panel, ask a question that cannot be answered by a single Cypher Template—a question whose query structure depends on the question itself (e.g. multi-hop aggregations, dynamic filters, or combinations your templates do not cover).

If you are using Northwind and want example questions:

* [copy]#Which customers ordered products from more than two different suppliers?#
* [copy]#Which products are in both the Beverages and Condiments categories?#
* [copy]#List the top three customers by total order value.#

Otherwise, ask an ad-hoc question that fits your graph. Run it and confirm you get a response.

== Step 2: Inspect the reasoning and generated Cypher

Expand the **Thought** section for the response. Find the **Applying agent tool** (or equivalent) entry for the Text2Cypher tool.

Check:

* **Generated Cypher**: Does it use the correct node labels and relationship types for your graph (e.g. for Northwind: `PLACED`, `CONTAINS`, `IN_CATEGORY`, `SUPPLIES`)?
* **Results**: Does the result set match what you expected? If the Cypher is wrong, the Text2Cypher tool description may be missing schema context.

image::../../1-introduction/lessons/2-create-with-ai/images/reasoning-text2cypher-detail.png[Text2Cypher tool detail showing the natural language input, the generated Cypher query, and the query results]

== Step 3 (optional): Improve the Text2Cypher description

If the generated Cypher used wrong relationship types or labels (for example `ORDERED_BY` instead of `PLACED`), edit the Text2Cypher tool:

. In the agent configuration, open the Text2Cypher tool (pencil icon).
. In the **Description** field, add or clarify the schema: list node labels and relationship types your graph uses.
. Save and click **Update agent**, then run the same question again and confirm the generated Cypher is correct.
