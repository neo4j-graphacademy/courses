[.question]
= How Parameters Work

A Cypher Template has a parameter `$category` with the description "The Northwind product category name, for example Beverages or Seafood". A user asks "Show me all Beverages". What value will the LLM pass for `$category`?

* [ ] `"category"`: it passes the parameter name as the value
* [ ] `null`: the LLM cannot extract values from short questions
* [x] `"Beverages"`: the LLM extracts the category name from the question using the parameter description
* [ ] `"all"`: the LLM interprets "all" as the qualifying word

[TIP,role=hint]
.Hint
====
The LLM reads the parameter description to understand what value to look for in the user's question. What word in "Show me all Beverages" matches "product category name"?
====

[TIP,role=solution]
.Solution
====
**"Beverages" is correct.**

The LLM reads the parameter description ("The Northwind product category name, for example Beverages or Seafood") to identify what type of value to extract. In the question "Show me all Beverages", the word "Beverages" matches the description, so it passes `$category = "Beverages"` to the query.

**Why the others are wrong:**

* `"category"`: The LLM passes the extracted value, not the parameter name.
* `null`: Short questions are not a problem. The LLM uses semantic matching to find the relevant word.
* `"all"`: "all" is a quantifier, not a category. The parameter description anchors the LLM to the category name.

**Recall:** The What is a parameter? section in this lesson explains that the description is an instruction to the LLM about what value to extract.
====
