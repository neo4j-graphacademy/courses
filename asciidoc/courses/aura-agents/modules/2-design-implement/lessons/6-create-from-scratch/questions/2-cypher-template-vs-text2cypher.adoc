[.question]
= Cypher Template or Text2Cypher?

Which of these questions should be handled by a Cypher Template rather than Text2Cypher?

* [x] "Get me details for product P123"
* [ ] "Which customers ordered products from more than 3 different suppliers?"
* [ ] "Show me customers whose order totals doubled compared to the previous quarter"
* [ ] "What were the most unusual purchasing patterns last month?"

[TIP,role=hint]
.Hint
====
Apply the test: can you write the complete Cypher right now, with only `$parameter` slots for variable values? Which question has a fixed MATCH pattern and only a variable value?
====

[TIP,role=solution]
.Solution
====
**"Get me details for product P123" should use a Cypher Template.**

The query is `MATCH (p:Product {id: $product_id}) RETURN p.name, ...`. The full structure is fixed and only the product ID changes. Write it once as a template.

**Why the others need Text2Cypher:**

* "More than 3 different suppliers": The filter count (3) could change, the multi-hop path through multiple relationship types is complex, and the aggregation structure varies. You cannot pre-define the `WHERE` and grouping logic.
* "Order totals doubled": Comparing totals across two time periods requires dynamic date calculations and aggregations that cannot be fully pre-written.
* "Most unusual patterns": "Unusual" has no fixed definition. The LLM needs to interpret and construct the query structure dynamically.

**Recall:** The Cypher Template vs. Text2Cypher NOTE in this lesson states: use a template when the `MATCH` pattern and `RETURN` clause are fixed, use Text2Cypher when the structure itself changes.
====
