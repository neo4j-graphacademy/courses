= Set Up Northwind
:order: 0
:type: lesson
== Introduction

The Northwind dataset is used in later lessons to build and test an agent. Setting it up now ensures you can run Cypher Template and Text2Cypher tools.

In this lesson, you will learn how to set up Northwind and enable tool authentication.


== Overview

You will complete three steps:

. Create an AuraDB instance
. Load the Northwind dataset
. Enable tool authentication so agents can query the instance


== Step 1: Create an AuraDB Instance

. Go to the link:https://console.neo4j.io/graphacademy[Aura Console^] and create a new AuraDB instance
. Name it something descriptive like "Northwind"
. Select your cloud provider and region

image::images/northwind-instance.png[Creating an AuraDB instance named Northwind]


== Step 2: Load Northwind Data

Use the Neo4j Data Importer to load the Northwind dataset. No API keys or embeddings required.

. Open your Northwind instance in the Aura Console and click **Import**
. Download the link:https://raw.githubusercontent.com/neo4j-graph-examples/northwind/main/import/northwind-complete.zip[northwind-complete.zip^] from the neo4j-graph-examples repo
. Click **Open Model (with data)** and select the downloaded zip file
. Run the import

The model includes Product, Category, Supplier, Customer, Order, and their relationships, for example Product PART_OF Category and Customer PURCHASED Order. This course uses **Cypher Template** and **Text2Cypher** tools only. No embeddings.

To verify the data loaded, open the **Query** tool and run: `MATCH (c:Customer) RETURN count(c) AS customers`. You should see 91 customers.


== Step 3: Enable Tool Authentication

Agents need permission to query your instance. Without this step, your instance will appear unavailable when you create an agent.

. Go to **Aura Console** → **Organization** → **Security Settings**
. Enable **Allow tools to connect with permissions from the user's project role**
. Select your Northwind instance from the list
. Verify **Generative AI Assistance** and **Aura Agent** are enabled; they should be on for most organizations
