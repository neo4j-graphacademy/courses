= Build an Agent with Tools
:order: 2
:type: challenge
:disable-cache: true

In this challenge, you will build a complete Northwind Analyst agent from scratch, add Cypher Template and Text2Cypher tools, and test it with real questions.


== Goal

Build an agent that can answer both predictable structured queries (using Cypher Templates) and ad-hoc questions (using Text2Cypher) over the Northwind knowledge graph.


== Before You Start

Make sure you have:

* A running Northwind AuraDB instance with tool authentication enabled
* Generative AI assistance and Aura Agent enabled in your Organization settings


== Steps

. **Create a new agent** in Aura Console → Data Services → Agents → Create Agent
. **Configure basics**: Set name to "Northwind Analyst", add a description, and connect to your Northwind instance
. **Write instructions**: Tell the agent its role and what it can do. Include node labels and relationship types so Text2Cypher generates valid Cypher. Example:
+
[source]
----
You are a Northwind retail analyst with access to a knowledge graph. The graph contains: Customer, Order, Product, Category, Supplier nodes. Relationships: PURCHASED (Customer→Order), PART_OF (Product→Category), SUPPLIES (Supplier→Product), ORDERS (Order→Product). Answer questions about customers, products, orders, and suppliers. Decline off-topic requests.
----
. **Add Cypher Template tools**: Create tools for Get Customer, Products by Category, and Top Customers by Order Count. Refer to the previous lesson for the tool configurations and Cypher queries
. **Add a Text2Cypher tool**: For ad-hoc questions you did not anticipate. Set a clear description that tells the agent when to use it
. **Save and test** with the questions below

See link:https://neo4j.com/docs/aura/aura-agent/[Aura Agent documentation^] for tool configuration details.


== Test Questions

Run these questions against your agent and check the **Thought** section to verify tool selection.

These should trigger a Cypher Template:

* "Which customers have placed the most orders?"
* "List products in the Beverages category"
* "Get details for customer QUICK"

These should fall back to Text2Cypher:

* "What are the top 5 products by revenue?"
* "Find customers who ordered products from more than 2 different suppliers"
* "Which categories have the highest average product price?"

This requires the agent to retrieve data and then reason over it:

* "Summarize what you know about customer ALFKI and their order history"


== What to Check

Verify that your agent:

* Selects a Cypher Template for the known question types
* Falls back to Text2Cypher for ad-hoc questions
* Returns accurate results that match what you would get from a direct Cypher query
* Handles relationship-heavy questions by traversing the graph
