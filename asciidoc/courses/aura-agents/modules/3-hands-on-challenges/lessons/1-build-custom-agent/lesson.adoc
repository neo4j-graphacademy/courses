= Build Your Own Custom Agent
:order: 1
:type: challenge
:disable-cache: true

In this challenge, you will build an Aura Agent that queries your knowledge graph. You can use the Northwind dataset from Module 2 or create an agent for your own AuraDB instance.


== Goal

Build a complete agent from scratch: configure name, instructions, and tools, then test it with real questions. Use Northwind or your own graph data.


== Choose Your Data Source

**Option A: Northwind**

If you completed Module 2, you already have Northwind loaded and the Northwind Analyst agent. You can skip to the next challenge, Publishing an Agent, or refine your agent here.

If you have not set up Northwind, complete link:../../2-design-implement/0-northwind-setup/[Set Up Northwind^] first. Then build an agent with Cypher Template and Text2Cypher tools as in link:../../2-design-implement/2-implement-simple-agents/[Implementing a Simple Aura Agent^].

**Option B: Your Own Aura Instance**

If you have an AuraDB instance with your own graph, build an agent that answers questions over your data. You need:

* A running AuraDB instance with a graph
* Tool authentication enabled for that instance
* A clear idea of what questions your agent should answer


== Before You Start

Make sure you have:

* A knowledge graph in a running AuraDB instance, Northwind or your own
* Tool authentication enabled; Generative AI Assistance and Aura Agent enabled in Organization settings


== Steps

. **Create a new agent** in Aura Console → Data Services → Agents → Create Agent
. **Configure basics**: Set name, description, and connect to your AuraDB instance
. **Write instructions**: Define the agent's role and scope. Include node labels and relationship types from your schema so Text2Cypher can generate valid Cypher
. **Add Cypher Template tools**: Create tools for common, predictable queries. Each tool needs a name, description, parameters, and Cypher query
. **Add Text2Cypher**, optional: For ad-hoc questions you cannot anticipate
. **Save and test** with questions that match your graph

See link:https://neo4j.com/docs/aura/aura-agent/[Aura Agent documentation^] for tool configuration details.


== Example: Northwind

If using Northwind, create tools such as:

* Get Customer: Return customer and recent orders for a customer ID
* Top Customers by Order Count: Return customers with the most orders
* Products by Category: List products in a category

Test with: "Which customers have placed the most orders?", "List products in the Beverages category", "Who has ordered Pavlova repeatedly?"


== Example: Your Own Graph

If using your own graph, design tools around your schema. For each tool:

* Name: Clear, descriptive
* Description: What it returns and when to use it; mention parameter names
* Parameters: Name, type, description so the LLM can extract values from user questions
* Cypher: The query with `$param` placeholders


== What to Check

Verify that your agent:

* Selects the right tool for each question
* Returns accurate results
* Handles relationship-heavy questions by traversing the graph
