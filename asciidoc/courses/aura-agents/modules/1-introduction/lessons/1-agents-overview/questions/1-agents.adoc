[.question]
= When to Use Aura Agents

Your team wants to query the knowledge graph in plain English from the Aura Console, without writing Cypher or application code. What does this lesson recommend?

Select the correct answer.

* [ ] A batch import job that loads CSV data into the graph
* [x] An Aura Agent connected to the graph so users can ask natural language questions
* [ ] Row-level security so each user sees only their data
* [ ] A parameterized Cypher API that the front end calls with an ID

[TIP,role=hint]
.Hint
====
The lesson says Aura Agents are useful when you want to expose knowledge graph data using natural language, without writing application code. Which option matches that?
====

[TIP,role=solution]
.Solution
====
**An Aura Agent connected to the graph.**

Aura Agents are a no/low-code platform where natural language questions turn into graph queries. You connect the agent to an AuraDB instance and users ask questions. The agent handles the rest. Batch import, row-level security, and a custom Cypher API are different use cases and require code or other tooling.
====
