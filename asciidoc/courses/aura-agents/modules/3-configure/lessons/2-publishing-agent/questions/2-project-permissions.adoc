[.question]
= Agent Project Permissions

Which project role is required to create, update, or delete an Aura Agent?

* [ ] Project Viewer
* [ ] Project Member
* [ ] Metrics Reader
* [x] Project Admin

[TIP,role=hint]
.Hint
====
The project permissions table in this lesson maps roles to agent actions. Which role has write access to agent configuration?
====

[TIP,role=solution]
.Solution
====
**Project Admin is correct.**

Project Admins can create, update, and delete agents. Viewers and Members can only list and invoke agents. They cannot change agent configuration or access settings.

**Why the others are wrong:**

* Project Viewer: Read-only access. Can list and invoke agents but cannot modify them.
* Project Member: Can invoke agents but cannot create, update, or delete them.
* Metrics Reader: Has access to performance metrics only, with no access to agent configuration.

**Recall:** The Project permissions section in this lesson defines the three relevant roles and their agent permissions.
====
