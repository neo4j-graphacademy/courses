= Course Summary
:pdf-summary:

Congratulations on completing _Context Graphs: Agent Memory with Neo4j_.

You connected short-term conversation memory, long-term knowledge with POLE+O, and reasoning traces in Neo4j using `neo4j-agent-memory`—so agent decisions become queryable, explainable data.

== Introduction to Agent Memory

You have learned:

* Why agent systems hit limits—and how context graphs address those gaps (_The Three Gaps_)
* How short-term, long-term, and reasoning memory work together (_The Three Memory Types_)
* Why Neo4j fits each memory layer better than row or vector-only stores (_Why Graphs for Agent Memory_)
* How to install and configure the library for your instance (_Installation and Configuration_)
* How session, message, entity, preference, fact, and reasoning APIs fit together (_The Complete Memory API_)

== Short-Term Memory

You have learned:

* What breaks when the model has no durable conversation state (_The Stateless Agent Problem_)
* The graph schema behind conversation history and what happens when messages are added (_Short-Term Memory_)
* How to create sessions, retrieve and search messages, and read the Cypher behind retrieval (_The Short-Term Memory API_)
* How `add_message` ties conversation text to long-term extraction (_How Messages Connect to Long-Term Memory_)

== Long-Term Memory

You have learned:

* What long-term memory must provide so every session does not start from zero (_The Persistent Knowledge Problem_)
* How to classify domain content with POLE+O and how entities and preferences are stored in the graph (_The long-term memory schema_)
* When vector search helps, when multi-hop graph queries matter, and how to combine both (_Why graphs over vector embeddings_)
* How the spaCy, GLiNER2, and LLM stages merge, deduplicate, and enrich entities (_Entity Extraction Pipeline_)
* How to store, search, and retrieve entities and facts with the long-term API (_The Long-Term Memory API_)

== Context Graphs and Reasoning Memory

You have learned:

* Why short- and long-term memory alone cannot answer full audit questions—and what reasoning memory adds (_The Auditability Problem_)
* What a context graph is and how entities, events, and context layers relate (_What is a context graph?_)
* The reasoning-memory schema and how it links across all three memory layers (_The reasoning memory schema_)
* How to record traces with Pydantic AI, context managers, and the low-level reasoning API (_Recording Reasoning Traces_)
* How to traverse tool calls, provenance, and similar past traces with Cypher (_Querying the Trace Graph_)
* How to run the end-to-end agent and verify the trace in the graph (_Build and run the agent_)

== Pro Tips

* Treat the reasoning graph as operational data: index and query patterns you will need in reviews and incident response, not only during development.
* Keep embedding and LLM provider configuration explicit in environment variables so sandboxes, staging, and production stay reproducible (_Installation and Configuration_).
* When tuning extraction, adjust merge and enrichment settings deliberately—small changes affect how entities deduplicate across sessions (_Entity Extraction Pipeline_).
