== Workshop Flow (2 hours)

=== Module 1: Introduction (15 min)

**Key points:**

* Show the recommendation query upfront (10 lines Cypher vs 38 lines SQL)
* Explain collaborative filtering: "What do people like me buy?"
* Demo the Aura console and Import tool briefly

**Hands-on:** Participants verify their sandbox credentials work.

---

=== Module 2: Foundation (45 min)

==== Lesson 1: Graph Elements (10 min) - THEORY

* Nodes = things (Customer, Product, Order)
* Labels = categories (:Customer)
* Relationships = connections (PLACED, CONTAINS)
* Properties = attributes (name, price)

**Instructor tip:** Keep this brief. The real learning happens when they import.

---

==== Lesson 3: Build Data Model with AI (10 min) - START HERE

**This is where participants design their graph model.** The lesson uses AI conversation to help identify nodes, relationships, and properties.

**How to run this:**

1. Have participants describe the Northwind problem: "product recommendations based on purchase history"
2. AI will suggest: Products, Customers, Orders as nodes
3. AI will identify relationships: PLACED, CONTAINS
4. Discuss the model before moving to import

**Instructor tip:** This sets up the mental model before hands-on import. Participants understand *why* they're creating each node type.

For advanced groups, let them try with their own data scenarios.

---

==== Lesson 4: Import Tool Overview (5 min) - INSTRUCTOR DEMO

**Live demo of the Import Tool UI.** The detailed UI tour slides are available for self-led learners but replaced with a live demo for instructor-led sessions.

Demo the three main areas:

1. **Data Sources tab** - Where you upload CSV files or connect to databases
2. **Graph Models tab** - The canvas where you build your model
3. **Import Jobs tab** - View import history and results

Also show the three-panel canvas layout (Data Sources | Model Canvas | Details Panel).

**Instructor tip:** Keep this quick (5 min). The hands-on import in the next lesson will reinforce all these concepts.

---

==== Lesson 4a: Import Data (20 min) - HANDS-ON DEMO

**This is the main guided walkthrough.** Students follow step-by-step.

The lesson loads ALL CSV files at once, then imports Products:

1. Download and load all 5 CSV files together
2. Review what each file contains (products, customers, orders, categories, order-details)
3. Map the Product node
4. Map columns to properties
5. Rename columns (productID → id)
6. Set the ID property
7. Preview and run the import

**Demo approach:**

1. Show each step slowly
2. Let participants follow along
3. Pause at checkpoints to let everyone catch up
4. Have them verify: "You should see 77 Product nodes created"

---

==== Lessons 4b-4c: Import Customers & Orders (15 min) - PRACTICE

**Challenge format:** Students work independently. Files are already loaded!

* Customer: 91 nodes from customers.csv
* Order: 830 nodes from orders.csv

**Walk around and help.** Common issues:

* Forgot to set ID property
* Wrong file selected
* Property names don't match

---

=== Module 3: Modeling Relationships (20 min)

==== Lesson 2: Import PLACED Relationships (20 min) - HANDS-ON

**First relationships!** Students create Customer→PLACED→Order.

Key steps:

1. Drag from Customer node to Order node in the canvas
2. Set relationship type: `PLACED`
3. Select source file: orders.csv
4. Map the foreign key: customerID → Customer.id
5. Run import: 830 PLACED relationships created

**Instructor tip:** This is where many students struggle. Demo it completely first, then let them try.

---

=== Module 4: Many-to-Many (20 min)

==== Lesson 2: Order-Product Relationships (15 min) - HANDS-ON

Create CONTAINS relationships with properties:

* Source: order-details.csv
* Properties: quantity, unitPrice, discount

"In relational, this would be a join table. In graphs, it's relationship properties."

---

=== Module 5: Recommendation Query (15 min)

==== Lesson 1: Build the Query (15 min) - HANDS-ON

Build step-by-step:

[source,cypher]
----
// Step 1: Find my products
MATCH (me:Customer {id: 'ALFKI'})-[:PLACED]->(:Order)-[:CONTAINS]->(myProduct:Product)

// Step 2: Find similar customers
MATCH (myProduct)<-[:CONTAINS]-(:Order)<-[:PLACED]-(other:Customer)
WHERE other <> me

// Step 3: Find their products I don't have
MATCH (other)-[:PLACED]->(:Order)-[:CONTAINS]->(rec:Product)
WHERE NOT (me)-[:PLACED]->(:Order)-[:CONTAINS]->(rec)

// Step 4: Rank by popularity
RETURN rec.name, count(DISTINCT other) AS recommendedBy
ORDER BY recommendedBy DESC
LIMIT 10
----

**Celebrate:** "You just built a recommendation engine in 10 lines!"

---

=== Wrap-up (5 min)

* Show the complete graph: `CALL db.schema.visualization()`
* Next steps: GraphAcademy courses, certification
* Q&A

== Delivery Tips

=== When to Demo vs. Let Them Do

|===
| Section | Approach

| Import Products
| Demo slowly, they follow along

| Import Customers/Orders
| They do it, you help

| Create Relationships
| Demo first, then they try

| Cypher queries
| Run together, discuss results
|===

=== Common Issues

|===
| Problem | Solution

| Lost credentials
| Reset password in Aura console, or share an instance

| Import fails
| Check ID property is set; verify file selection

| Query returns nothing
| Check property names (case-sensitive); verify import completed

| Fell behind
| Share your model JSON file
|===

=== Participant Checkpoints

After each section, verify everyone is at the same state:

* After Products: `MATCH (p:Product) RETURN count(p)` → 77
* After Customers: `MATCH (c:Customer) RETURN count(c)` → 91
* After PLACED: `MATCH ()-[r:PLACED]->() RETURN count(r)` → 830
* After CONTAINS: `MATCH ()-[r:CONTAINS]->() RETURN count(r)` → 2155

'''

== Data Model Reference

=== Core (Required)

----
(:Customer)-[:PLACED]->(:Order)-[:CONTAINS]->(:Product)-[:IN_CATEGORY]->(:Category)
----

=== Relationship Properties

----
[:CONTAINS {quantity, unitPrice, discount}]
----

=== Node Counts

* 91 Customers
* 830 Orders
* 77 Products
* 8 Categories
* ~4,000 relationships total
