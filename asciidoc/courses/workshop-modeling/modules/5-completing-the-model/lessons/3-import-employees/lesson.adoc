== Importing Employees

You have Customers, Orders, and Products. Now add Employees to track which employee processed each order.

In this challenge, you will work independently to import Employee nodes and create SOLD relationships connecting employees to orders.


[.slide]
== Download employees.csv

[NOTE]
.Additional data source
====
This optional lesson uses `employees.csv`, which is not included in the main workshop files.

Download it separately: link:https://data.neo4j.com/northwind/employees.csv[employees.csv^]
====


[.slide]
== File structure: employees.csv

The employees.csv file contains employee information:

[options="header"]
|===
| Column | Type | Description
| employeeID | Integer | Unique identifier for each employee
| lastName | String | Employee's last name
| firstName | String | Employee's first name
| title | String | Job title
| hireDate | Date | Hire date
| city | String | City location
|===

This file has **9 rows** - one for each employee.


[.slide.col-2]
== Understanding the SOLD relationship

[.col]
====
**Business rule:** Each order was processed by an employee.

The `employeeID` column in orders.csv references the employee who handled each order.
====

== Challenge: Create an Employee node mapping

Using what you've learned about the Import tool:

1. Map employees.csv to an Employee node
2. Rename `employeeID` to `id` (for consistency)
3. Include relevant properties: firstName, lastName, title, hireDate, city


[.slide]
== Challenge: Create SOLD relationships

1. Map orders.csv as the relationship source
2. Create relationship type: `SOLD`
3. **From:** Map `employeeID` in orders.csv to Employee node's `id`
4. **To:** Map `orderID` in orders.csv to Order node's `id`
5. Run the import


[.slide]
== Expected results

After importing, you should have:

* **9 Employee nodes** created
* **830 SOLD relationships** created (one for each order)


[.slide]
== Verify your import

Run this query to check your results:

[source,cypher]
.Check sales assignments
----
MATCH (e:Employee)-[:SOLD]->(o:Order)
RETURN e.firstName + ' ' + e.lastName AS employee,
       e.title AS title,
       count(o) AS ordersSold
ORDER BY ordersSold DESC
----
