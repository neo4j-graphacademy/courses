== Validate Results

Once you have imported the Employee nodes and created both REPORTS_TO and SOLD relationships, verify your results.

verify::[]

[TIP,role=hint]
.Hint
====
Make sure you completed all three parts:

**Check:**

* Employee node mapping created with employeeID as unique identifier
* REPORTS_TO relationship created using employees.csv as source
  - From: employeeID → Employee node
  - To: reportsTo → Employee node (self-referential)
* SOLD relationship created using orders.csv as source
  - From: employeeID (in orders.csv) → Employee node
  - To: orderId (in orders.csv) → Order node
* Import executed successfully
====

[TIP,role=solution]
.Solution
====
Run these queries to check your Employee nodes and relationships:

[source,cypher]
.Check Employees and relationship counts
----
MATCH (e:Employee)
WITH count(e) AS employeeCount
MATCH ()-[r1:REPORTS_TO]->()
WITH employeeCount, count(r1) AS reportsToCount
MATCH ()-[r2:SOLD]->()
RETURN employeeCount, reportsToCount, count(r2) AS soldCount;
----

You should see 9 employees, 8 REPORTS_TO relationships, and 830 SOLD relationships.

[source,cypher]
.Verify organizational hierarchy
----
MATCH (e:Employee)
WHERE NOT (e)-[:REPORTS_TO]->()
RETURN e.firstName + ' ' + e.lastName AS ceo, e.title;
----

You should see one CEO with no REPORTS_TO relationship.

**If verification fails:**

* Verify employees.csv and orders.csv files were uploaded
* Check Employee node has employeeID as unique identifier
* For REPORTS_TO: both From and To should map to Employee nodes (self-referential)
* For SOLD: From maps to Employee, To maps to Order
* Ensure you're using the correct CSV file for each relationship (employees.csv for REPORTS_TO, orders.csv for SOLD)
====
