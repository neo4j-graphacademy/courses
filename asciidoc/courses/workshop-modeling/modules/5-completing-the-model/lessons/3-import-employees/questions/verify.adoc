== Validate Results

Once you have imported the Employee nodes and created SOLD relationships, verify your results.

verify::[]

[TIP,role=hint]
.Hint
====
Make sure you completed both parts:

**Check:**

* Employee node mapping created with `id` as unique identifier (renamed from employeeID)
* SOLD relationship created using orders.csv as source
  - From: employeeID (in orders.csv) → Employee node's `id` property
  - To: orderID (in orders.csv) → Order node's `id` property
* Import executed successfully
====

[TIP,role=solution]
.Solution
====
Run this query to check your Employee nodes and relationships:

[source,cypher]
.Check Employees and SOLD relationship count
----
MATCH (e:Employee)
WITH count(e) AS employeeCount
MATCH ()-[r:SOLD]->()
RETURN employeeCount, count(r) AS soldCount
----

You should see 9 employees and 830 SOLD relationships.

[source,cypher]
.Verify top sellers
----
MATCH (e:Employee)-[:SOLD]->(o:Order)
RETURN e.firstName + ' ' + e.lastName AS employee, count(o) AS orders
ORDER BY orders DESC
LIMIT 5
----

**If verification fails:**

* Verify employees.csv and orders.csv files were uploaded
* Check Employee node has `id` as unique identifier (renamed from employeeID)
* For SOLD: From maps employeeID → Employee's id, To maps orderID → Order's id
* Ensure you're using orders.csv for the SOLD relationship
