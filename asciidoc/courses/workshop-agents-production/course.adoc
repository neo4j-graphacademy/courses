= Zero to Agents in Production (1 hour)
:status: draft
:duration: 1 hour
:caption: Load Northwind with Open Model (with data), validate with Cypher, build an Aura agent, then practice snapshots, logs, and access control.
:key-points: Import with Open Model (with data), Cypher smoke tests, Aura agents, snapshots and restore, query logs, security and RBAC
:usecase: operations
:categories: workshops, coredb, aura

This hands-on workshop compresses the path from loading data to running Aura agents in production into a single hour.

You will:

* Load the Northwind graph data model using the **Open model (with data)** workflow in the Import tool.
* Run short Cypher queries in the Query tool to verify the graph.
* Create an Aura AI agent with **Create with AI** and test it.
* Take a snapshot, restore when needed, and locate query logs and security-related settings (including role-based access control at a high level).

== Duration

1 hour

== Prerequisites

* A Neo4j Aura instance you can import into (for example link:/courses/aura-fundamentals/[Aura Fundamentals^] if you need a first-time setup).
* AuraDB Professional or higher is recommended for full access to scheduled snapshots, exports, and some log features, as described in link:/courses/aura-administration/[Aura In Production^].

== What you will learn

* How to open a packaged graph model with data in Aura
* How to read counts and paths with Cypher
* How to create and try an Aura agent on your graph
* How to protect data with snapshots and how to find logs and access controls in the console
