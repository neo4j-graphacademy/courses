== Introduction

Use the Query tool to prove the Northwind import looks correct before you hand the graph to an agent or an application.

You will:

. Count nodes by label.
. Traverse a short path from customer to product.
. Run one aggregation for orders.

[.slide]
== Open Query

Open the *Query* tool and connect to the instance that received the import.

console::Open Query Tool[tool=query]

[.slide]
== Count customers

[source, cypher, role=noplay]
----
MATCH (c:Customer)
RETURN count(c) AS customers
----

[.slide]
== Count products

[source, cypher, role=noplay]
----
MATCH (p:Product)
RETURN count(p) AS products
----

[.slide]
== Path check

Follow purchases from customers to products.

[source, cypher, role=noplay]
----
MATCH (c:Customer)-[:PURCHASED]->(o:Order)-[:ORDERS]->(p:Product)
RETURN c.companyName, o.orderID, p.productName
LIMIT 10
----

If counts look wrong, compare relationship types to the diagram from the Import lesson. The reference Northwind package uses `PURCHASED` and `ORDERS`.

[.slide]
== Aggregation

Customers with the most orders:

[source, cypher, role=noplay]
----
MATCH (c:Customer)-[:PURCHASED]->(o:Order)
RETURN c.companyName, count(o) AS orders
ORDER BY orders DESC
LIMIT 10
----
