== Introduction

The Graph Data Science in Practice Workshop is designed to provide participants with a foundational and holistic set of skills to perform graph data analyses on real-world data.

The workshop introduces basic graph data modeling concepts, guides users through a repeatable workflow across GDS in the Neo4j Browser, GDS via the Python Client and includes an introduction to Aura Graph Analytics. 

Along their journey, participants will encounter deep dives into several basic algorithms across five main categories: Centrality, Community Detection, Pathfinding, Node Embeddings and Similarity. 

The workshop can be presented online or in person and is best suited for 4-hour sessions. If time-constrained, instructors can teach only module 1 & 2, eschewing the Python and Aura Graph Analytics sections entirely,

== Prerequisites

Learners should have a fundamental understanding of graph databases and Neo4j. A rudimentary understanding of basic Cypher is sufficient. 

Learners will need a GraphAcademy and github account—if needed, these can be created during the workshop.

If delivering module 4 'Aura Graph Analytics', learners will require an Aura Pro Trial. Ideally, this account should be created before the session begins. 

== Learning objectives

* The fundamentals of Graph Data Science and the GDS Project → Run → Write workflow
* How to create graph projections for different analytical questions
* Community detection algorithms (Louvain, WCC) to uncover fraud risks
* Centrality algorithms (PageRank, Betweenness, Degree) to rank influential nodes
* Pathfinding algorithms (Dijkstra, Yen's) to optimize supply chain routes
* Node embeddings (FastRP) for machine learning features
* How to build complete citation analysis pipelines with the Python GDS client
* How to run graph analytics at scale with Aura Graph Analytics

== Outline plan

(All timings are estimates and adaptable)

=== Introduction

(10 mins)

Introduce yourself, the session, and have participants login

Use the introduction slides to introduce yourself and the agenda

Introduce GraphAcademy and ask learners to create an account and goto the workshop URL in the notes document.

https://graphacademy.neo4j.com/courses/workshop-gds/

Starter activity - "ask people what (if any) experience they have had with Neo4j, GDS or graph data science in general"

Show learners the GraphAcademy course summary, summarise the learning objectives and enrol in the course.

=== GDS Foundations

(60 mins)

Explain the core concepts of graph data science and the GDS workflow.

This module frontloads as much theory as possible so users can get into the practical use-cases. Ensure that, by the end, everyone has a basic understanding of the concepts explored.

* Distinguish between GDS (the Neo4j plugin) and graph data science (the discipline). Introduce the key access points (Desktop, Aura, Python)
* Teach the Project -> Run -> Write workflow.
* Walk through the five algorithm categories. Frame each by the question it answers. Stress that algorithms compute scores -- you decide what those scores mean
* Explain graph structures: monopartite, bipartite, multipartite, and heterogeneous.
* Projection Practice is the hands-on exercise. Give participants 5 minutes per exercise. If short on time, shorten the time windows or skip this lesson -- its purpose is to build habitual comfort with projections and provide a safe space to fail. There is little practical difference between 5 minutes and 2 minutes per exercise.
* Run algorithms from all five categories on the actor collaboration network. Don't dwell on algorithm details here.
* Cover the five execution modes (stream, stats, mutate, write, estimate). 
* Teach projection configuration: undirected relationships, weights, and node properties.

=== Community Detection for Fraud 

(45 mins)

Apply community detection and centrality algorithms to uncover fraud risks in a real P2P financial transactions dataset

This module should move quickly. The theoretical lessons (Louvain deep dive, WCC deep dive) provide the deepest algorithm coverage in the workshop -- spend time here. 

The practical lessons recontextualise the algorithms in a real-world use case; they are not intended to reteach from scratch. Do not treat the use-case framing as 'best practices' or how-to guides. They are not that. Their only purpose is recontextualization for later domain transfer. They are, at most, 'playbooks'. 

Consider saving questions for the end of each lesson. Many questions have been preempted by the content, so don't get tied up answering questions that will be answered within two slides.


=== GDS Python Client

(60-70 mins)

Move from the Browser to the Python GDS client on a citation network

For sessions under 4-hours, demonstrate the notebooks, but do not expect participants to run them with you.

For all sessions, take a break before you start this one. Ensure that, before the break, learners click the Codespace button before the break. This should ensure environments are setup in time to start. 

This module contains the most sections, but it should not take much longer than the first.

It comes in two sections:

. Introduction to the Python Client
. Algorithms on a dataset (same structure as M2)

Do not hang around on the intro to the Python client sections. They will see and run these queries and commands multiple times.

Spend a little more time on the Cypher -> Native projection section. Ensure they have grasped the differences before moving on.

Most of the slides in this module exist only to complement the notebooks. Every code block in the notebooks is represented on a slide, with images used to visualize particularly difficult concepts, where possible. Ensure you are using the correct slides with the correct notebooks.

In terms of algorithms, this module covers quite a few:

* PageRank (essential)
* Betweenness (essential)
* Louvain (already covered earlier, but now in a new context. Do not spend too long on this, and focus only on the contextual differences, as outlined in the content)
* FastRP – node embeddings, generally, are tough for people to understand. Even their descriptions are often hard to interpret. Make sure learners understand that a lack of understanding is expected, and that it is okay.


=== Aura Graph Analytics

(30-40 mins) + (10-15 mins setup)

Run analytics workloads in the cloud with ephemeral sessions with pathfinding examples

For sessions under 4 hours, or where you are running over time, skip this module, and encourage learners to continue on their own time. 

Bear in mind, this module requires users to set up their own AuraDB with a Pro Trial and to correctly configure their .env. This could take upwards of 10-15 mins.

Introduces Aura Graph Analytics.

Shows users how to create AGA sessions and use remote projection. 

Covers Pathfinding with Dijkstra's and Yen's.

If you need to chop some time out, just cover the technical basics and the Movies dataset lessons — encourage them to run the other notebooks on their own time.
