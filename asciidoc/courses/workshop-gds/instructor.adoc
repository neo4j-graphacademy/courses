== Introduction

The Graph Data Science in Practice Workshop is designed to provide participants with a foundational and holistic set of skills to perform graph data analyses on real-world data.

The workshop introduces basic graph data modeling concepts and guides users through a repeatable workflow using GDS in the Neo4j Browser. Participants will apply community detection and centrality algorithms to a real-world fraud detection use case.

Along their journey, participants will encounter an introduction to the five algorithm categories (Centrality, Community Detection, Pathfinding, Node Embeddings and Similarity) followed by deep dives into Community Detection and Centrality for fraud detection.

The workshop can be presented online or in person and is best suited for 4-hour sessions.

For an extended experience covering the Python GDS client and Aura Graph Analytics, see the companion link:/courses/workshop-gds-python-aga/[GDS Python Client & Aura Graph Analytics^] workshop.

== Prerequisites

Learners should have a fundamental understanding of graph databases and Neo4j. A rudimentary understanding of basic Cypher is sufficient.

Learners will need a GraphAcademy account—if needed, this can be created during the workshop.

== Learning objectives

* The fundamentals of Graph Data Science and the GDS Project → Run → Write workflow
* How to create graph projections for different analytical questions
* Community detection algorithms (Louvain, WCC) to uncover fraud risks
* Centrality algorithms (PageRank, Betweenness, Degree) to rank influential nodes
* How to encode domain hypotheses as graph relationships for targeted analysis

== Outline plan

(All timings are estimates and adaptable)

=== Introduction

(10 mins)

Introduce yourself, the session, and have participants login

Use the introduction slides to introduce yourself and the agenda

Introduce GraphAcademy and ask learners to create an account and goto the workshop URL in the notes document.

https://graphacademy.neo4j.com/courses/workshop-gds/

Starter activity - "ask people what (if any) experience they have had with Neo4j, GDS or graph data science in general"

Show learners the GraphAcademy course summary, summarise the learning objectives and enrol in the course.

=== GDS Foundations

(60 mins)

Explain the core concepts of graph data science and the GDS workflow.

This module frontloads as much theory as possible so users can get into the practical use-cases. Ensure that, by the end, everyone has a basic understanding of the concepts explored.

* Distinguish between GDS (the Neo4j plugin) and graph data science (the discipline). Introduce the key access points (Desktop, Aura, Python)
* Teach the Project -> Run -> Write workflow.
* Walk through the five algorithm categories. Frame each by the question it answers. Stress that algorithms compute scores -- you decide what those scores mean
* Explain graph structures: monopartite, bipartite, multipartite, and heterogeneous.
* Projection Practice is the hands-on exercise. Give participants 5 minutes per exercise. If short on time, shorten the time windows or skip this lesson -- its purpose is to build habitual comfort with projections and provide a safe space to fail. There is little practical difference between 5 minutes and 2 minutes per exercise.
* Run algorithms from all five categories on the actor collaboration network. Don't dwell on algorithm details here.
* Cover the five execution modes (stream, stats, mutate, write, estimate).
* Teach projection configuration: undirected relationships, weights, and node properties.

=== Community Detection for Fraud

(45 mins)

Apply community detection and centrality algorithms to uncover fraud risks in a real P2P financial transactions dataset

This module should move quickly. The theoretical lessons (Louvain deep dive, WCC deep dive) provide the deepest algorithm coverage in the workshop -- spend time here.

The practical lessons recontextualise the algorithms in a real-world use case; they are not intended to reteach from scratch. Do not treat the use-case framing as 'best practices' or how-to guides. They are not that. Their only purpose is recontextualization for later domain transfer. They are, at most, 'playbooks'.

Consider saving questions for the end of each lesson. Many questions have been preempted by the content, so don't get tied up answering questions that will be answered within two slides.
