[.verify.slide]
= Validate the Merge

Once you have run all three steps, click the **Check Database** button to verify.

verify::[]

[TIP,role=hint]
.Hint
====
The check fails if parts were duplicated - `MERGE (p:Part {partNumber: ...})` must use exactly the same key property the Module 2 load used.

**Check:**

* 30 vehicles and 39 work orders loaded
* Still exactly 11 `Part` nodes (no duplicates)
* `REPLACED` relationships connect work orders to parts
====

[TIP,role=solution]
.Solution
====
Run this query to inspect the merge:

[source,cypher]
.Check the merged graph
----
RETURN COUNT { (:Vehicle) } AS vehicles,
       COUNT { (:WorkOrder) } AS workOrders,
       COUNT { (:Part) } AS parts,
       COUNT { (:Section)-[:REFERENCES_PART]->(:Part)<-[:REPLACED]-(:WorkOrder) } AS bridges
----

You should see 30 vehicles, 39 work orders, exactly 11 parts, and a non-zero bridge count.

**If verification fails:**

* If `parts` is greater than 11, you created duplicates - delete them with `MATCH (p:Part) WHERE p.name IS NOT NULL AND NOT (p)<-[:REFERENCES_PART]-() DETACH DELETE p` and re-run Step 1 using `partNumber` as the merge key
* Complete the Module 2 challenges first if `bridges` is 0 and there are no `Section` nodes
====
