[.slide]
== Your sandbox

This workshop uses an integrated Neo4j sandbox - no setup required.
The sandbox window is on the right of the screen, and it includes the Graph Data Science library you will use in Module 3.

The database starts **empty**.
That is deliberate: building each shape yourself - load, link, detect, merge - is the workshop.

Run this query in the sandbox to confirm you are connected:

[source,cypher]
.Confirm your sandbox is ready
----
RETURN gds.version() AS gdsVersion
----

You should see a version number, confirming Graph Data Science is available.


[.slide]
== The two halves of the lakehouse

In production, AutoFix's data lives in Databricks:

[cols="1,2,2", options="header"]
|===
| Half | Where it lives | What's in it
| Documents
| PDFs in cloud storage
| 3 service manuals, 5 technical service bulletins, 2 recall notices
| Warehouse
| Delta tables under Unity Catalog (`autofix.service.*`)
| 30 vehicles, 39 work orders, 11 parts, 13 procedures, 6 trouble codes
|===

For this workshop, both halves are published as CSV exports, so you can load them with `LOAD CSV` and spend the time on graph value, not connector plumbing.
The PDFs have already been parsed: each document's sections, and the part numbers and trouble codes mentioned in each section, are extracted into tables.

[NOTE]
.Production import paths
====
In production you would not export CSVs by hand.
The take-home repository includes the Databricks workflow for this import - Spark connector for Delta tables, a parsing pipeline for the PDFs - plus adapters for Snowflake and BigQuery.
Module 4 shows where it slots in.
====

== The shared keys

Two identifier types appear on _both_ halves:

* **Part numbers** - `IC-2042-A` appears in the Falcon manual, in bulletin TSB-21-114, in recall RC-2021-04, _and_ in the warehouse's parts catalog and work order line items
* **Diagnostic trouble codes (DTCs)** - `P0301` appears in manual diagnosis sections, in bulletins, _and_ on diagnosed work orders

Nothing in the lakehouse treats these as links.
Everything you build in the next three modules does.
