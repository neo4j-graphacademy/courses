[.verify.slide]
= Validate the Themes

Once you have written the communities back to the graph, click the **Check Database** button to verify.

verify::[]

[TIP,role=hint]
.Hint
====
Stream mode (Steps 2 and 3) does not change the database - only Step 4 (`gds.leiden.write`) does.

**Check:**

* The projection `doc-links` was created in Step 1
* You ran `gds.leiden.write` with `writeProperty: 'communityId'`
====

[TIP,role=solution]
.Solution
====
Run this query to check the written communities:

[source,cypher]
.Check community assignments
----
MATCH (s:Section)
RETURN s.communityId AS theme, count(*) AS sections
ORDER BY sections DESC
----

Every section should have a theme, with the largest containing 9 sections.

**If verification fails:**

* If the projection already exists, drop it first: `CALL gds.graph.drop('doc-links')`
* Re-run Step 1 and Step 4
====
