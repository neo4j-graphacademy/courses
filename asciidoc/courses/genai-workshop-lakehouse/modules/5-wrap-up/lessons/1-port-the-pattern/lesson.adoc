== Introduction

You built three shapes on one fictional repair chain.
None of them depend on auto repair - or on Databricks.

In this lesson, you will see how to port the shapes to your own lakehouse and where each one slots into an agent stack.


[.slide]
== Swap the connector, keep the shapes

Everything after the load step was pure graph:

[cols="1,2", options="header"]
|===
| Layer | What changes when you move platform
| Source (Databricks, Snowflake, BigQuery, ...)
| The connector: Spark connector, Snowflake external function, BigQuery export - the take-home repository includes adapters
| Load
| Column names in the `LOAD CSV` / connector mapping
| **Shapes: trees, communities, paths**
| **Nothing**
|===

The Cypher you wrote for trees, Leiden, and cross-boundary traversal does not reference Databricks once.


[.slide]
== Re-skin the domain

The pattern fits any business with **manuals plus assets** - documents that describe things, and records of what happened to those things, sharing IDs:

* **Manufacturing** - equipment manuals + maintenance logs, joined on model and error codes
* **Pharma** - protocols and regulatory filings + batch records, joined on compound and lot numbers
* **Financial services** - policies and procedures + transactions and cases, joined on product and account identifiers
* **IT operations** - runbooks and postmortems + incidents and deploys, joined on service names and error codes

The questions port too: "what does the documentation say applies to _this_ asset, and what actually worked on assets like it?"


[.slide]
== Where the shapes slot into an agent stack

Each shape becomes a tool the agent can call - alongside vector search, not instead of it:

* **Trees** → a `navigate_docs` tool: fetch a table of contents, walk to a section, list what a document covers. Grounded browsing instead of similarity guessing
* **Themes** → a `get_themes` tool: a six-card summary of the whole estate that fits in one prompt, for "where are the patterns?" questions
* **Paths** → parameterized traversal tools like the comeback query: deterministic multi-hop answers where Text2SQL fails silently

Vector search still handles "find me a passage about X" - and works better here, because every hit carries its tree position and theme as context.
