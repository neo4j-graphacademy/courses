== Course Description

Your agent can reach your data but still can't use it reliably.
Vector search and Text2SQL each hand it a slice, but not the view to know what is truly relevant and how to connect the right information.
Without that, answers come back confident but wrong.
That is not a model or query problem - it is a context problem, and thinking in terms of *shapes* is what cracks it.

In this hands-on workshop, you will build three reusable graph shapes from lakehouse data using Neo4j, so an agent can navigate and view the right context:

* **Table of Contents (Trees)** - navigate what's there
* **Themes (Communities)** - surface patterns nobody named
* **Connections (Paths)** - trace how entities, documents, and records relate

You will work with data from AutoFix Group, a fictional national auto-repair chain on a Databricks lakehouse: service manuals, bulletins, and recall notices as PDFs in cloud storage, and Vehicle, WorkOrder, Part, and Procedure Delta tables under Unity Catalog.
The two halves share part numbers and diagnostic trouble codes - and that overlap is what you will build on.

You build the shapes the way you would on the job: in a hosted Codespace, you and your coding agent author a *service-advisor skill* - tools and policy, module by module - and finish by handing it a live work order: the agent picks the evidence-backed fix, catches an open recall, orders the part through an API, and leaves an auditable trail in the graph.
Every query is also runnable by hand in the integrated sandbox.

At the heart of the skill is the multi-hop traversal no vector index or Text2SQL tool can answer reliably - from a service bulletin on the PDF side to repair outcomes on the table side.

The patterns are Databricks-first but portable: swap the connector and the same shapes work on Snowflake, BigQuery, or anywhere your data lives.

== Prerequisites

Before taking this workshop, you should have:

* A basic understanding of graph databases and Neo4j
* The ability to read and run basic Cypher queries
* Familiarity with data warehouse or lakehouse concepts (tables, keys)

We recommend completing the link:/courses/neo4j-fundamentals/[Neo4j Fundamentals^] course first.
No Graph Data Science experience is required.

For the hands-on path as designed, you will need a coding agent such as Claude Code, Cursor, Codex, or Gemini CLI, and a terminal to install link:https://github.com/neo4j-labs/neo4j-cli[neo4j-cli^].
Without one, every challenge can be completed in the integrated sandbox instead.

== What you will learn

* Why agent context is a problem of shapes, not queries - and where vector search and Text2SQL fall short
* How to build an agent skill - tools, scripts, and policy - that a coding agent runs through the Neo4j CLI
* How a pipeline turns real PDFs and warehouse tables into one graph with shared-key links
* How to run Leiden community detection to surface repair themes nobody named
* How to build judgment tools whose multi-hop Cypher crosses the document-to-table boundary
* How an agent decides and acts on policy - grounding, evidence, recalls, escalation - with an auditable trail
* How to port the pattern to another lakehouse or domain
