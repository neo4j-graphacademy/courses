== 很久很久以前……
**要了解我们是如何到达现在的，我们首先需要回到过去。**

1736 年，普鲁士的柯尼斯堡。

image::images/konigsberg.jpg[普鲁士柯尼斯堡的插图地图]

image::images/7-bridges.jpg[哥尼斯堡七桥,role=right]

莱昂哈德·欧拉大概正坐在他的办公桌前，左右为难。
康斯贝格（今俄罗斯加里宁格勒）被普雷格尔河分为四个部分，由七座桥梁连接。

欧拉正在思考的问题是：我们能不能步行穿过这座城市，只需穿过七座桥中的每座桥一次？

=== 图论基础

他最终通过将问题重新表述解决了这个问题，并以此奠定了图论的基础。

// image::images/abstraction.jpg[问题的抽象,role=right]

他意识到，陆地本身并不是一个重要因素。事实上，连接陆地的桥梁才是最重要的。

他的方法是用抽象的术语定义问题，将每个陆地表示为抽象的 *顶点* 或 _节点_ ，然后将这些陆地与一组代表桥梁的七个 *边* 连接在一起。这些元素组成了一个“图”。

=== 应用理论

image::images/graph.jpg[柯尼斯堡七桥问题的图表示,role=right]

使用这种抽象，欧拉能够明确地证明这个问题没有答案。无论你从哪里进入这个图，以及你以何种顺序过桥，你都无法在没有至少两次过桥的情况下到达每个陆地。

但这并不是完全白费。
虽然图起源于数学，但它们也是一种非常方便的建模和分析数据的方式。
虽然我们的数据肯定有价值，但真正能够增加价值的是数据之间的联系。
创建或推断记录之间的关系可以对数据集产生真正的洞察力。

快进 300 年，这些初始原则广泛用于解决复杂问题，包括路线查找、供应链分析和实时推荐。
