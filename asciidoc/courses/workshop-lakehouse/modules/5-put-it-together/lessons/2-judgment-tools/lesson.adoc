== Challenge

The navigation and theme tools tell the agent what the documents _say_.
The judgment tools tell it what actually _worked_ - by federating: grounding in the document graph, then reading the repair history from live BigQuery.

These two are written from specification, the way you would build them on the job: the spec is the contract, your coding agent is the pair programmer, and the data tells you if you got it right.


[.slide]
== The specs

Open `skill/SKILL.md` and find the **Judgment and actions** section.
Two of its four specs are this challenge:

* **`what_fixed_this.py <vin> <code>`** - ground in Neo4j (documents covering the code and the parts they name), then read outcomes from BigQuery (uses and comebacks for those parts on similar vehicles), join and rank; the first row is the evidence-backed fix
* **`recall_exposure.py <vin>`** - recalls in scope for the vehicle's model (Neo4j) whose remedy part is current and this vehicle has never received (BigQuery)

These scripts **do not exist yet** - creating them is the challenge.
Hand each spec to your coding agent, review what it writes, and test against the questions below.
`skill/scripts/db.py` runs Neo4j, `skill/scripts/bq.py` runs BigQuery; the full specs and the graph schema are at the top of SKILL.md.


[.slide]
== Why this query needed the merged graph

Whichever way your agent writes `what_fixed_this`, it must cross the boundary: it starts in document trees (which documents cover this code, which parts do they name) and ends in warehouse history (which work orders replaced that part, did the vehicle come back) - and those two halves live in different stores.
The bridge is the **shared key**: the part number the document names is the same string the BigQuery rows use. And the BigQuery join itself - work orders to vehicles to parts - is grounded by the connections graph from Module 2, so the SQL is right.

No vector index can return this, and Text2SQL guesses the joins. Federation on the key, grounded by the connections shape, is why this works.
