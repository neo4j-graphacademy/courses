== Find every document about a part

Dani's real question: the recall mentioned coil `IC-2042-A` - what else covers it?

Part numbers and trouble codes are not entity nodes - they live in the section text.
So you find every document about a part with **full-text search**, then walk up the `HAS` tree to the document that owns each matching section:

[source,cypher]
.All documents that mention a part
----
CALL db.index.fulltext.queryNodes('content_search', '"IC-2042-A"')
YIELD node, score
WHERE node:Section
MATCH (doc:Document)-[:HAS*]->(node)
RETURN DISTINCT doc.area AS type, doc.title AS document,
       collect(node.displayName) AS sections
----

The hits span all three folders - `bulletins`, `manuals`, and `recalls`: the misfire bulletins (`tsb-20-501`, `tsb-22-301`), the coil recall (`rc-2021-01`), and the ignition manuals all mention the same coil.
That cross-document view is exactly what 40 keyword hits buried.

**Try experimenting:**

* Change the term to `"BP-7720"` to map the brake-pad documents
* Run the same question through the tool: `python skill/scripts/search.py '"IC-2042-A"'`

== Follow the links

From the misfire bulletin, what related guidance did its author point to?

[source,cypher]
.What does tsb-20-501 link to?
----
MATCH (b:Document {id: 'tsb-20-501'})-[:HAS*]->(:Section)
      -[:LINKS_TO]->(tgt)
MATCH (d:Document)-[:HAS*0..]->(tgt)
RETURN DISTINCT d.title AS linkedDocument,
       tgt.displayName AS section
ORDER BY linkedDocument
----

The first `[:HAS*]` walks down into the bulletin's own sections; `LINKS_TO` jumps to whatever each one references; the second `[:HAS*0..]` walks back up to the document that owns the target (length `0` handles a link that lands on a document directly).
The bulletin points into the Falcon misfire-diagnosis procedure and into the Heron ignition manual's coil-identification, coil-replacement, and spark-plug sections.
An agent following these links retrieves a **connected set** - the right meaning in the right shape.
