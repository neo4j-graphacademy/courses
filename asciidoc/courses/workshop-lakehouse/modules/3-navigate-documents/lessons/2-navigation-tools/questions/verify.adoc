[.verify.slide]
= Validate the Graph

Your tools build on the loaded graph - click the **Check Database** button to confirm both halves are in place.

verify::[]

[TIP,role=hint]
.Hint
====
The load pipeline builds everything: `python load/load_documents.py` in your Codespace.
It is safe to re-run - every write is a `MERGE`.

**Check:**

* Your `.env` points at this sandbox (the credentials from Module 1)
* The pipeline printed "Graph ready" with 183 documents and 985 sections
====

[TIP,role=solution]
.Solution
====
Run the pipeline from the Codespace terminal:

[source,shell]
----
python load/load_documents.py
----

Then check the counts:

[source,cypher]
----
RETURN COUNT { (:Document) } AS documents,
       COUNT { (:Section) } AS sections,
       COUNT { ()-[:LINKS_TO]->() } AS links
----

You should see 183 documents, 985 sections, and roughly 1300 links.

**If verification fails:**

* Confirm `.env` has the sandbox credentials from Module 1, then re-run the pipeline
* The pipeline rebuilds the document half cleanly on every run and leaves Module 2's connections graph (`:Table`/`:Column`) in place - just re-run it. Avoid a blanket `MATCH (n) DETACH DELETE n`; that would also drop the connections graph the finale needs
====
