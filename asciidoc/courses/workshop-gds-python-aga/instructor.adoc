== Introduction

The GDS Python Client & Aura Graph Analytics Workshop is designed to take participants from Browser-based GDS to programmatic graph analytics with Python and scalable cloud-based analytics with Aura Graph Analytics.

This workshop is a companion to the link:/courses/workshop-gds/[Graph Data Science in Practice^] workshop, which covers GDS foundations and fraud detection using GDS in the Neo4j Browser.

The workshop can be presented online or in person and is best suited for 4-hour sessions.

== Prerequisites

Learners should have completed the link:/courses/workshop-gds/[Graph Data Science in Practice^] workshop, or have equivalent experience with GDS concepts, projections, and the Project → Run → Write workflow.

Learners will need:

* A https://graphacademy.neo4j.com[GraphAcademy] account
* A https://github.com[GitHub] account (for Codespace access)
* An AuraDB account with a Pro Trial (for Module 2: Aura Graph Analytics)

Ideally, the Aura Pro Trial account should be created before the session begins.

== Learning objectives

* How to use the Python GDS client for graph analytics workflows
* Native projections as an alternative to Cypher projections
* Deep dives into PageRank, Betweenness Centrality, and Louvain on citation data
* Node embeddings (FastRP) for machine learning features
* How to run graph analytics at scale with Aura Graph Analytics
* Pathfinding with Dijkstra's and Yen's K-Shortest Paths for logistics optimization

== Outline plan

(All timings are estimates and adaptable)

=== Introduction

(10 mins)

Introduce yourself, the session, and have participants login.

Introduce GraphAcademy and ask learners to create an account and goto the workshop URL.

https://graphacademy.neo4j.com/courses/workshop-gds-python-aga/

Starter activity — confirm participants have GDS experience (either from the prerequisite workshop or their own background). Briefly recap the Project → Run → Write workflow.

=== GDS Python Client

(60-70 mins)

Move from the Browser to the Python GDS client on a citation network.

Ensure that learners click the Codespace button early. The environment takes approximately 10 minutes to configure.

This module contains the most sections, but it should not take much longer than the first module in the prerequisite workshop.

It comes in two sections:

. Introduction to the Python Client
. Algorithms on a dataset

Do not hang around on the intro to the Python client sections. They will see and run these queries and commands multiple times.

Spend a little more time on the Cypher → Native projection section. Ensure they have grasped the differences before moving on.

Most of the slides in this module exist only to complement the notebooks. Every code block in the notebooks is represented on a slide, with images used to visualize particularly difficult concepts, where possible. Ensure you are using the correct slides with the correct notebooks.

In terms of algorithms, this module covers quite a few:

* PageRank (essential)
* Betweenness (essential)
* Louvain (already covered in the prerequisite workshop, but now in a new context. Do not spend too long on this, and focus only on the contextual differences, as outlined in the content)
* FastRP — node embeddings, generally, are tough for people to understand. Even their descriptions are often hard to interpret. Make sure learners understand that a lack of understanding is expected, and that it is okay.


=== Aura Graph Analytics

(30-40 mins) + (10-15 mins setup)

Run analytics workloads in the cloud with ephemeral sessions with pathfinding examples.

Bear in mind, this module requires users to set up their own AuraDB with a Pro Trial and to correctly configure their .env. This could take upwards of 10-15 mins.

Introduces Aura Graph Analytics.

Shows users how to create AGA sessions and use remote projection.

Covers Pathfinding with Dijkstra's and Yen's.

If you need to chop some time out, just cover the technical basics and the Movies dataset lessons — encourage them to run the other notebooks on their own time.

=== Close

Q&A and call to action

Remind learners they will have access to the workshop content indefinitely.

Run a Q&A session.

Suggest GraphAcademy courses for learners to continue their learning.
