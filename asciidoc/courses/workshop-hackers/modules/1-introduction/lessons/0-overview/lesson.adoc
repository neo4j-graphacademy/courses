== Overview

In this lesson, you will learn what you are building today, how the session is structured, and what tools you have available.

In this hackathon, you will build a GraphRAG application — an AI system that uses a knowledge graph to give LLMs accurate, connected, contextual answers.

[.slide]
== Understanding the session structure

The next 30 minutes are a rapid fundamentals session. Then you hack.

[cols="1,3",options="header"]
|===
| Module | What you will do

| *Introduction (this module)*
| Learn why graphs improve LLM accuracy, design your own data model, create mock data, and write your first Cypher queries

| *Hack!*
| Build your GraphRAG application using your AI coding agent — powered by the GraphAcademy MCP server

|===

[.slide]
== Using your tools

You have three things working for you today:

. **Your Neo4j sandbox** — a blank graph database provisioned just for you, waiting for your data
. **Your coding agent** (GitHub Copilot, Claude, Cursor, or similar) — an AI assistant in your Codespace
. **The GraphAcademy MCP server** — connected to your coding agent, giving it deep Neo4j expertise

You don't need to know Cypher perfectly or have used Neo4j before. Your coding agent knows how to help, and the GraphAcademy MCP server has the answers when it gets stuck.

In the next lesson, you'll set up the MCP server so your coding agent is ready to go.
