= Workshop Summary
:pdf-summary:

Congratulations on completing the **Email PDFs to Knowledge Graph** workshop!

In 4 hours, you've gone from raw email PDFs to structured, parsed records -- extracting text, handling OCR noise, and building hybrid parsing pipelines.

== What You Built

You've learned to:

* Extract text from PDF documents using PyMuPDF with OCR fallback
* Parse unstructured email text into structured records (From, To, Cc, Subject, Date)
* Build rule-based parsers that handle OCR noise and layout variation
* Use LLMs for parsing with token-efficient prompt strategies
* Combine approaches into a hybrid pipeline for production use

== What You Produced

Your parsing pipeline now produces structured email records with:

* Sender, recipients, and CC fields extracted from noisy PDF text
* Subject lines and dates normalized across layout variants
* A hybrid strategy: rule-based templates for the majority, LLM fallback for the rest

== Key Takeaways

=== Extraction is the Foundation

* OCR quality directly impacts everything downstream
* File size is a useful proxy for text quality (clean vs garbled)
* Always validate extraction output before parsing

=== Parsing Requires Multiple Strategies

* Template-based parsing handles structural variation better than pure regex
* LLMs handle edge cases but cost more -- use them as a fallback, not default
* A hybrid pipeline balances cost, speed, and accuracy
