[.question]
= Greedy extraction vs templates

The greedy extractor silently absorbed `Ce:` (OCR for `Cc:`) into the To field. How does the template approach handle this differently?

* [ ] Templates also absorb Ce: into To, but flag it as low confidence
* [ ] Templates correct Ce: to Cc: automatically using spell checking
* [x] The template for emails without Cc rejects the file because Ce: appears where Subject: is expected; the Cc template matches because it recognizes Ce: as an OCR variant of Cc:
* [ ] Templates skip any line they don't recognize

[TIP,role=hint]
.Hint
====
Templates encode exact role sequences. What happens when the sequence doesn't match -- does the template guess, or does it refuse?
====

[TIP,role=solution]
.Solution
====
Templates are rejection tools. The no-Cc template (`alt_FStX`) expects Subject: after To: but finds Ce: -- it returns None. The Cc template (`alt_FStCX`) expects a Cc label after To: and recognizes Ce: as an OCR variant -- it matches correctly. The greedy extractor had no mechanism to reject, so it silently produced wrong data.
====
