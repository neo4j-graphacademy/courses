[.question]
= Template coverage

The three starter templates cover ~98.6% of the corpus. What happens to the remaining ~70 emails?

* [ ] They are discarded from the dataset
* [ ] The template parser guesses the closest match
* [x] They are collected as failures for the LLM to handle in a later notebook
* [ ] The parser retries with different boilerplate stripping

[TIP,role=hint]
.Hint
====
Templates return None when no template matches. What does the pipeline do with None results?
====

[TIP,role=solution]
.Solution
====
When no template matches, the email is added to the failures list. These failures are sent to an LLM in notebook 2.6 via the Batch API. The hybrid pipeline in notebook 2.7 merges both template and LLM results into a single dataset.
====
