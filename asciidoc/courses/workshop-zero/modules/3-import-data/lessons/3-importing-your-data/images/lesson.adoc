== Import the Northwind dataset

You need to link:https://raw.githubusercontent.com/neo4j-graph-examples/northwind/refs/heads/main/import/northwind-customer.zip[download the Northwind dataset^] and create a new blank graph data model:

. link:https://raw.githubusercontent.com/neo4j-graph-examples/northwind/refs/heads/main/import/northwind-customer.zip[Download the empty Northwind dataset^]
. Open the *Import* tool in Aura and select *Graph Models*. 
+
console::Open Import Tool[tool=import]
+
image::images/aura-import-graph-annotated.png["Aura Console - Import - Graph Models"]
. Create a new graph data model.
+
image::images/new-graph-model.png["Blank graph model in the Import tool"]
. Use the `...` menu to `Open model (with data)` and select the `northwind-customer.zip` file you downloaded.
+
image::images/open-model-with-data-annotated.png["... menu - Open model with data"]
. You will see the tables (csv files) from the Northwind dataset in the left-hand panel, and a single Customer node in the graph data model.
+
image::images/northwind-data-set.png["The northwind dataset, files and graph data model"]

[.transcript-only]
== Nodes

The `Customer` node has already been created for you. 

image::images/customer-node.png["Customer node in the graph data model"]

You are going to create the `Order` nodes: 

. Create a new *Node label* - `Order`
+
image::images/new-node-label-annotated.png["+ Node Label button highlighted"]
. Add the `Order` as the *Label*
. Select `orders.csv` as the *Table*
. Use the *Map from table* function to create the properties:
+
image::images/map-from-table-annotated.png["Map from table button highlighted"]
. Select all the columns, except the `customerId` and `employeeId`.
+
image::images/order-columns.png["All columns selected except customerId and employeeId"]
+
When you confirm the mapping, the properties, their data types, and column will be automatically mapped. 
+
image::images/order-properties.png["Properties mapped from the table columns"]
. The `orderId` property should be selected as the unique ID.
. *Run the import* to create the `Customer` and `Order` nodes.

View the orders in the graph:

[source, cypher, role=noplay]
.Order nodes
----
MATCH (o:Order)
RETURN o
LIMIT 100
----

image::images/query-order.png["Result of the query showing the Order nodes"]

[.transcript-only]
== Relationships

You can now create the `(Customer)-[:PURCHASED]->(Order)` relationship.

. Drag a relationship from the `Customer` node to the `Order` node.
+
image::images/create-relationship.gif["Creating a relationship by dragging an arrow from the source node to the target node"]
. Set the *Relationship Type* to `PURCHASED`
. Select `orders.csv` as the *Table*
. The Node ID mapping tells Import how to connect the nodes.
** Select `customerID` from the the `Customer` node
** Select `orderID` from the `Order` node
+
image::images/purchased-relationship.png["The setup of the purchased relationship, showing the node ID mapping"]
. Run the import

When you have added the `Order` nodes and `PURCHASED` relationships you can view the customer's orders:

[source, cypher, role=noplay]
.Customer orders
----
MATCH (c:Customer)-[p:PURCHASED]->(o:Order)
RETURN c,p,o
LIMIT 100
----

image::images/purchased-query.png["Result of the query showing customers, their purchased relationships and the orders"]

This process of adding nodes and relationships to the graph data model, and mapping them to source data, can be repeated to continue to extend the graph with more data.

== Challenge

Your task is to add the `Product` nodes and `ORDERS` relationship to `Order` nodes.

[source, mermaid]
.Order ORDERS Product
----
include::diagrams/order-orders-product.mermaid[]
----

To include the product nodes you will need to:

. Add a new `Product` node label importing from `products.csv`
. Map the properties from the table
. Set the `productId` as the unique ID.

To create the relationship from `Order` to `Product`:

. Drag a new relationship from `Order` to `Product` with type `ORDERS`
. Use `order-details.csv` as the source table.
. Use the `orderID` and `productID` for the node ID mapping.
. Map the `unitPrice`, `quantity` and `discount` as relationship properties.

When you have added the customer's purchased products, you can view the customers complete product orders:

[source, cypher, role=noplay]
.Customer orders
----
MATCH (c:Customer)-[p:PURCHASED]->(o:Order)-[or:ORDERS]->(pr:Product)
RETURN c,p,o,or,pr
LIMIT 100
----
