== AI generated dashboard

You can create a new dashboard using the *Create with AI* feature:

. Open the *Dashboards* tool
+
console::Open Dashboards[tool=dashboards]

. Select *Create with AI*.
+
image::images/create-with-ai-annotated.png["The Create with AI button highlighted"]

. Enter a description of the dashboard you want to create, such as [copy]#Metrics related to Customers and Orders#.
+
image::images/prompt.png["The prompt entered to create the dashboard"]

. The dashboard will be generated based on your description.
+
image::images/ai-dashboard.png["A new dashboard generated by the AI assistant with multiple cards showing different metrics related to customers and orders"]

Explore the dashboard and the information it contains.

=== Create a dashboard card

You can add additional information to your dashboard by adding new *cards*.

[NOTE]
.Cards
====
Dashboard cards are individual components that display specific data visualizations.
====

You are going to add a card that shows the graph of a specific customer's orders:

. Click **Add a card** to create a new visualization.
+
image::images/add-card-annotated.png["Add card button in the bottom right highlighted]

. Change the card title from "New card" to [copy]#Customer Orders Overview#.

. Select **Graph** from the visualization type dropdown
+
image::images/new-card-annotated.png["New card with the title, visualization type, and query filled in"]

. Use the following Cypher query in the Query field:
+
[source,cypher,role=noplay]
----
MATCH (c:Customer)<-[r:PURCHASED]->(o:Order)
WHERE c.companyName = 'Eastern Connection'
RETURN c,r,o
----
+
[TIP]
.Previewing the card results
====
You can preview the results of the card query before saving it by clicking the *Preview* button.
====

. *Save* the card.

Your new card will be added to the bottom of dashboard, showing the orders for the customer "Eastern Connection".

image::images/new-card-on-dash-annotated.png["The new card added to the dashboard showing the graph of the customer's orders"]

=== Making cards interactive with filters

Filters allow you to dynamically change what data is displayed without modifying the cards.

You can create a filter that lets users select what customer they want to view the orders for:

. Select **Add filter**.
+
image::images/add-filter.png["Add filter button highlighted"]

. Edit the filter name from "New filter" to [copy]#Customer Filter#.

. Create a filter with the following settings:

* Filter type: `Node property`
* Node label: `Customer`
* Property key: `companyName`

image::images/filter-annotated.png["Filter settings"]

. Add a default value for the filter, such as `Alfreds Futterkiste`.

. *Save* the filter.

The filter will be added to the dashboard, allowing you to select different customers from the dropdown.

image::images/new-filter-on-dash-annotated.png["The new filter added to the dashboard with the default value selected"]

=== Link the filter to the card

You need to update the card query to use the filter parameter:

. Edit the *Customer Orders Overview* card.
+
image::images/edit-filter-annotated.png["Edit card button highlighted"]

. Update the query to use the new filter parameter.
+
[source,cypher,role=noplay]
----
MATCH (c:Customer)<-[r:PURCHASED]->(o:Order)
WHERE c.companyName = $customer_companyname
RETURN c,r,o
----

. Save the card.

image::images/card-and-filter-annotated.png["Customer Orders Overview card with filter linked"]

Updating the filter selection will now update the card results, allowing you to explore different customers and their orders.

Experiment by modifying other cards on the dashboard to use the filter.
