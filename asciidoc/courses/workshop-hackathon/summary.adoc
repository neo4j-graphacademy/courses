= Course Summary

Congratulations on completing the GraphRAG Hackathon.

You have learned:

* Why graph databases make LLMs more accurate — and what GraphRAG is
* How Neo4j stores data as nodes, relationships, and properties
* How to design a graph data model for your own use case
* How to write Cypher queries to create and traverse connected data
* How to use the GraphAcademy MCP server to build with your coding agent

You have also built something — a GraphRAG application connected to a real Neo4j database.
Keep the momentum going.

== Continue learning

=== Neo4j and Cypher

* link:https://graphacademy.neo4j.com/courses/neo4j-fundamentals/[Neo4j Fundamentals^] — a one-hour introduction to graph databases and the Neo4j property graph model
* link:https://graphacademy.neo4j.com/courses/cypher-fundamentals/[Cypher Fundamentals^] — learn to read and write Cypher, Neo4j's query language, in one hour
* link:https://graphacademy.neo4j.com/courses/modeling-fundamentals/[Graph Data Modeling Fundamentals^] — learn how to design graph data models that answer the right questions

=== GraphRAG and Generative AI

* link:https://graphacademy.neo4j.com/courses/genai-fundamentals/[Neo4j and Generative AI Fundamentals^] — a two-hour course covering RAG, GraphRAG, vector indexes, and building AI applications with Neo4j
* link:https://graphacademy.neo4j.com/courses/llm-vectors-unstructured/[Introduction to Vector Indexes and Unstructured Data^] — learn how to combine vector similarity search with graph traversal for richer retrieval

=== Building with the MCP server

* link:https://graphacademy.neo4j.com/courses/genai-mcp-neo4j-tools/[Developing with Neo4j MCP Tools^] — go deeper with the tools you used today: `neo4j_graph_modeler`, `neo4j_query_builder`, `neo4j_mock_data_generator`, and more
