Take five minutes here before you start hacking — the goal is to leave this lesson with data in your database and a query that proves your graph can answer the question you care about.

A graph data model is only as good as the questions it can answer.
Before you generate data, decide what your application will need to know.
Good graph questions involve following relationships across multiple hops:

* "What products do people like me tend to buy?" — collaborative filtering
* "Who in my network connects me to this person?" — shortest path
* "Which accounts are part of a suspicious transaction ring?" — fraud detection
* "What topics are experts in my field writing about?" — knowledge discovery

Use this conversation to refine your data model around your question, generate realistic mock data, and get Cypher scripts you can run in your Neo4j sandbox.
Your coding agent and the GraphAcademy MCP server can do the same thing — use whichever feels faster.

[.conversation-options]
* I have a data model — help me check it can answer my question
* I want to build a recommendation engine
* I want to explore connections in a network
* I want to detect fraud or anomalies in transaction data
* Let me describe my use case from scratch
