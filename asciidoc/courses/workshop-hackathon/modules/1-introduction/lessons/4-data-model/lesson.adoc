= Build your data model
:type: conversation
:order: 5
:duration: 5

// Source: Adapted from neo4j-fundamentals/modules/1-graph-thinking/lessons/4-build-data-model
// and workshop-modeling/modules/2-foundation/lessons/3-build-your-data-model

Spend about 5 minutes designing your model here. You can continue refining it during the hack session with your coding agent.

Start by describing the problem you are trying to solve, or the data you are working with. You can describe it in plain language, share your database schema, or paste a few lines from a CSV file.

I can help you:

* Identify the **nodes** — the distinct entities in your domain
* Assign **labels** — the categories those nodes belong to
* Identify the **relationships** — how those entities connect
* Identify the **properties** — the attributes that describe nodes and relationships

Where should we start?

[.conversation-options]
* I want to build a recommendation engine based on user behavior
* I want to analyze connections in a social or professional network
* I want to detect fraud or anomalies in transaction data
* I have my own use case — let me describe it
