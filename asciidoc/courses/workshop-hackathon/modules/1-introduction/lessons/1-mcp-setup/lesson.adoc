== Introduction

Before you start building, you need two things: a development environment and a database connection.

In this lesson, you will learn how to open your GitHub Codespace, connect it to your Neo4j sandbox, and understand what tools the GraphAcademy MCP server makes available to your coding agent.

== Your Neo4j database

GraphAcademy has provisioned a blank Neo4j sandbox for you.
Use the credentials below to connect your coding agent to your database.

[source,subs="verbatim,attributes"]
.Database credentials
----
NEO4J_URI="{instance-scheme}://{instance-ip}:{instance-boltPort}"
NEO4J_USERNAME="{instance-username}"
NEO4J_PASSWORD="{instance-password}"
NEO4J_DATABASE="{instance-database}"
----

Copy these into the `.env` file in your Codespace. The file already exists — just fill in the values.

== The GraphAcademy MCP server

An MCP server (Model Context Protocol server) is a standard way to give an AI coding agent a set of callable tools — functions it can invoke on your behalf during a conversation.

The GraphAcademy MCP server is available at `https://mcp.graphacademy.neo4j.com` and is already configured in the Codespace.
Your coding agent will automatically have access to tools for:

* *Learning* — browse GraphAcademy courses, lessons, and enrolments
* *Asking questions* — get grounded answers about Neo4j from lesson content
* *Modeling* — design graph data models for your use case
* *Building* — generate Cypher queries, import scripts, mock data, and project scaffolding

[.transcript-only]
====
Your coding agent has access to the following tools:

[cols="1,3",options="header"]
|===
| Tool | Description

| `list_courses`
| List all active GraphAcademy courses

| `list_lessons`
| List all lessons in a course grouped by module

| `list_enrolments`
| List all courses the user is enrolled in with progress

| `enrol`
| Enrol the authenticated user in a course

| `get_next_lesson`
| Get the next incomplete lesson (optionally marking the current one complete)

| `mark_lesson_complete`
| Mark a lesson complete and return the next lesson's content

| `verify_lesson`
| Run the verification Cypher for a challenge lesson against the user's Neo4j database

| `answer_question`
| Ask a question about Neo4j; returns a grounded answer from lesson content

| `neo4j_graph_modeler`
| Design a graph data model (labels, relationships, properties, indexes)

| `neo4j_query_builder`
| Build Cypher queries; can introspect live schema and run EXPLAIN/PROFILE

| `neo4j_import_advisor`
| Get advice and scripts for importing data into Neo4j

| `neo4j_mock_data_generator`
| Generate realistic seed/test data as Cypher, CSV, or JSON

| `neo4j_project_builder`
| Build a Neo4j application with driver setup and project structure

|===
====
