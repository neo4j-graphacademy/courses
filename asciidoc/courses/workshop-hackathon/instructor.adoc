= GraphRAG Hackathon: Facilitator Guide

https://graphacademy.neo4j.com/courses/workshop-hackers/[graphacademy.neo4j.com/courses/workshop-hackers/]

== Introduction

The GraphRAG Hackathon is a 30-minute fundamentals primer followed by an open-ended hacking session.
Participants learn just enough about GraphRAG, Neo4j, and Cypher to get started, then use an AI coding agent — connected to the GraphAcademy MCP server — to build something of their own.

The format is intentionally lightweight. You are not delivering a lecture; you are setting context, facilitating the conversation lessons, and then circulating during the hack.

The session can run from 90 minutes to a full day, depending on the hack time available.

== Prerequisites

=== For the instructor

What you should know before delivering this workshop:

* https://graphacademy.neo4j.com/courses/neo4j-fundamentals/[Neo4j Fundamentals]
* https://graphacademy.neo4j.com/courses/cypher-fundamentals/[Cypher Fundamentals]
* https://graphacademy.neo4j.com/courses/llm-fundamentals/[Introduction to Vector Indexes and Unstructured Data]
* https://graphacademy.neo4j.com/courses/genai-workshop/[Neo4j and Generative AI Workshop]

You should have completed the workshop yourself before delivering it, including running through the Codespace environment and using the MCP server with a coding agent.

=== For participants

* A GitHub account (for GitHub Codespaces)
* Basic programming experience (Python or TypeScript)
* An OpenAI API key (or another supported LLM provider key)

No prior Neo4j or graph database knowledge is required.

== Learning objectives

By the end of the 30-minute instructed session, participants will:

* Understand why graph databases improve LLM accuracy (GraphRAG)
* Know the four building blocks of a Neo4j graph (nodes, labels, relationships, properties)
* Have designed a graph data model for their own use case
* Understand basic Cypher syntax (MATCH, MERGE, indexes, constraints)
* Have mock data loaded in their sandbox and seen example graph traversal queries

By the end of the hack session, participants will have built (or made meaningful progress on) a GraphRAG application of their own choosing.

== Preparation

What you need before the session:

. *A copy of the repository* — https://github.com/neo4j-graphacademy/workshop-hackers[github.com/neo4j-graphacademy/workshop-hackers] — review the README, the starter code, and the MCP server configuration
. *A working Codespace* — open one yourself to verify the environment is set up correctly (MCP server connected, `.env` template in place)
. *Your own API key* — test that your coding agent can use the GraphAcademy MCP server tools before the session
. *Enrol in the course* — https://graphacademy.neo4j.com/courses/workshop-hackers/ — go through it once, then delete your enrolment so you see the same view as participants on the day
. *Introduction slides* — prepare a personalised slide to introduce yourself and the session agenda (optional; the course overview lesson covers the agenda)

Share the following with participants at the start:

* The course URL: https://graphacademy.neo4j.com/courses/workshop-hackers/
* The repository URL: https://github.com/neo4j-graphacademy/workshop-hackers
* Your preferred coding agent (recommend GitHub Copilot, Claude, or Cursor — all support MCP)

== Outline plan

All timings are estimates and adaptable.

=== Welcome and setup (15 min)

*Get everyone into their environment before the slides start.*

Ask participants to:

. Go to the course URL and enrol
. Open the Codespace (the button is in Lesson 1: Getting Started)
. Copy their sandbox credentials from Lesson 1 into the `.env` file
. Run the environment check from the README to confirm everything is connected

While they do this, circulate and help anyone who is stuck. The most common issue is GitHub sign-in for Codespaces — make sure everyone has a GitHub account before the session.

Do not start the slides until the majority have their Codespace open, their credentials in place, and the environment check passing. This prevents the embarrassing moment where the hack starts but half the room can't connect to Neo4j.

=== Module 1: Introduction (30 min)

*The instructed content — present as a group, then let participants explore.*

Go through the lessons with the group. The slide-based lessons (Why GraphRAG?, What is Neo4j?, Cypher Basics) are designed to be presented, not read individually. Use the GraphAcademy course slides or your own slides if you prefer.

[cols="1,1,4",options="header"]
|===
| Lesson | Time | Facilitator notes

| Welcome (L0)
| 3 min
| Set the scene. What will they build? Why Neo4j? Confirm everyone is enrolled.

| Getting Started (L1)
| 5 min
| Make sure everyone has their `.env` populated. Ask them to run the environment check described in the repository README — a passing result confirms the Neo4j credentials and API key are correctly set. Then test the MCP server by asking the agent: _"What tools do you have available?"_ — it should list the GraphAcademy MCP tools.

| Why GraphRAG? (L2)
| 7 min
| Present the slides. The key point: graphs store connections, and connections are what make LLMs more accurate. Use the recommendation example — it's the one that lands best with audiences.

| What is Neo4j? (L3)
| 7 min
| Present the slides. Slow down on the node/label/relationship/property building blocks — this is the most important conceptual content. The modeling decision (node vs property) will come up during the hack; prime the audience for it now.

| Build Your Data Model (L4)
| 5 min
| This is an AI-driven conversation lesson. Ask participants to spend 5 minutes describing their domain to the AI and seeing what data model it suggests. Circulate and look at what people are building — this gives you context for helping during the hack. If the room is hesitant, suggest "product recommendations" as a safe starter.

| Cypher Basics (L5)
| 7 min
| Present the slides. The "Try it now" section at the end has three queries. If participants have data in their sandbox (from L4), ask them to run these now. If not, reassure them they will work after L6.

| Create Mock Data (L6)
| 5 min
| Another AI-driven conversation lesson. The AI will generate Cypher to load data and demo queries. Ask participants to run the demo queries in Neo4j Browser — seeing their data in the graph for the first time is the "aha" moment of the instructed section. Celebrate it.

|===

=== Module 2: Hack! (open-ended)

*Step back. Participants lead. You facilitate.*

Once everyone has moved to the Hack Time lesson, your role changes. Participants decide what to build; the coding agent and MCP server guide them.

Your job during the hack:

* Circulate and ask "what are you building?" — this keeps energy up and surfaces blockers early
* When someone is stuck, help them prompt their coding agent better rather than solving it yourself
* Call out interesting projects to the room — hearing what others are building sparks ideas
* Keep track of time and give a "15 minutes left" warning if there is a scheduled close

Suggested starter prompts to share with participants:

----
"Use neo4j_project_builder to scaffold a Python GraphRAG app connected to my Neo4j database."

"Use neo4j_query_builder to write a query that finds the top 5 most connected nodes in my graph."

"Use answer_question to explain how to implement a vector index for semantic search in Neo4j."
----

=== Close

* Ask 2–3 participants to show what they built (even partial work counts)
* Remind participants they keep their GraphAcademy sandbox access
* Suggest next steps on GraphAcademy:
** https://graphacademy.neo4j.com/courses/genai-workshop/[Neo4j and Generative AI Workshop]
** https://graphacademy.neo4j.com/courses/llm-fundamentals/[Introduction to Vector Indexes and Unstructured Data]
** https://graphacademy.neo4j.com/courses/cypher-fundamentals/[Cypher Fundamentals]

== Common issues

[cols="1,2",options="header"]
|===
| Issue | Resolution

| Codespace won't open
| Check the participant has a GitHub account and is signed in. Free monthly Codespace allowance must not be exhausted.

| MCP server not visible to coding agent
| Verify the `.mcp.json` config in the Codespace root. Restart the coding agent if the config was added after launch.

| Sandbox credentials not working
| Credentials are shown on Lesson 1 of the course. Confirm the participant is enrolled and has a sandbox provisioned (usually takes <60 seconds after enrolment).

| Coding agent writes incorrect Cypher
| Direct the participant to use the `neo4j_query_builder` or `answer_question` tools explicitly rather than relying on the agent's general knowledge.

| API key not working
| Run the environment check from the README — it will confirm which variables are missing or invalid. Confirm the key is in the `.env` file and the Codespace was restarted after editing `.env`. Check the provider dashboard for rate limits.

|===
