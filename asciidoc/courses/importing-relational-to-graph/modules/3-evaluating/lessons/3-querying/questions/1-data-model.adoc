[.question]
= SQL and Cypher Performance

For which type of query does Cypher typically outperform SQL?

* [ ] Simple aggregations on a single table
* [ ] Bulk insert operations
* [x] Queries requiring multiple JOINs (for example, 4 or more tables)
* [ ] Mathematical computations


[TIP,role=hint]
.Hint
====
Cypher follows relationship pointers directly; SQL JOINs are computed at query time and cost increases as more tables are joined. Multi-hop or multi-table JOIN queries are where graphs excel.
====

[TIP,role=solution]
.Solution
====
**Cypher outperforms SQL for queries requiring multiple JOINs**, for example when four or more tables are joined.

Why?
* SQL JOINs are computed at query time, requiring index lookups for each join
* Cypher traversals follow pre-stored relationship pointers; each hop is a direct follow
* As JOIN count increases, SQL performance degrades while Cypher cost grows with the number of hops, not with total table size

SQL may be faster for:
* Simple single-table aggregations without traversal benefit
* Bulk operations with mature optimization
* Mathematical computations with more mature functions
====
