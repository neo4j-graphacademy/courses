[.question]
= Validation Test Cases

Which of the following should be included in a data validation test plan after importing relational data into Neo4j? Select all that apply.

* [x] Verify node counts match source table row counts
* [x] Check that relationship counts match expected foreign key connections
* [ ] Verify that all SQL queries still run unchanged in Neo4j
* [ ] Spot-check only one record; no need to compare counts

[TIP,role=hint]
.Hint
====
Validation should check node and relationship counts against the source; property types and constraint enforcement catch wrong types and duplicate records. You use Cypher in Neo4j, not SQL.
====

[TIP,role=solution]
.Solution
====
**Node count validation** and **relationship count validation** belong in the plan.

* **Node counts** — Compare Neo4j node counts per label to source row counts so you catch missing or duplicated data.
* **Relationship counts** — Confirm the number of relationships matches expected connections from foreign keys.

**SQL queries** do not run in Neo4j; you validate with Cypher. **Spot-checking one record** is not enough; compare full counts and sample queries.
====
