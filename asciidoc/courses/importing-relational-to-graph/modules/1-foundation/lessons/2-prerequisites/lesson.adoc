= Prerequisites and how to measure a meaningful import
:order: 1

== Introduction

To import relational data into Neo4j, you need a Neo4j instance, your own source data (or an optional sample), and the Data Importer tool. The course focuses on schema analysis and modeling for any relational source; PostgreSQL, BigQuery, Snowflake, and CSV exports are examples, not requirements.

In this lesson, you will complete a prerequisites checklist and define how you will measure that your import was meaningful. You will use these same criteria in the final module to validate your data.


== Prerequisites checklist

Before you start the hands-on work, confirm you have the following.

* A Neo4j AuraDB Professional instance (no credit card required)
* Your own relational data or CSV export (or the optional Northwind CSV files—see below)
* Basic understanding of relational databases: tables, rows, columns, foreign keys
* Basic understanding of graph databases: nodes, relationships, properties
* Familiarity with SQL queries
* Basic knowledge of Cypher query language

== Using the Data Importer

The Data Importer is built into the AuraDB console; for Sandbox or Self-Managed Neo4j, use `data-importer.neo4j.io`. Use your own CSV files or the optional Northwind CSVs from the previous section.


=== Open the Data Importer

. In Neo4j Aura, open your AuraDB Professional instance and click **Import** in the left sidebar
. For Sandbox or Self-Managed: open `data-importer.neo4j.io` and connect with your instance credentials


=== Add your data

The Data Importer accepts local files only (no direct URL or remote file option). To add your data:

. Use CSV (or TSV) exports from your own database, or if you are using the Northwind example: download from link:https://github.com/neo4j-graph-examples/northwind/tree/main/import[neo4j-graph-examples/northwind^] and unzip locally if needed.
. In the Data Importer, click **New data source**, then choose **.CSV/.TSV (local)**.
. Upload each CSV file (or drag and drop into the Files panel). For Northwind you would use files such as `customers.csv`, `orders.csv`, `products.csv`, etc.; for your own data, use the exports that match your schema.

[NOTE]
.Zip upload not supported
====
The Data Importer does not accept zip files directly; you must unzip and upload individual CSV files. If this is a blocker for your workflow, consider requesting a zip-upload option via the Neo4j feedback or feature-request channels.
====

== How will you measure that the import was meaningful?

Define success criteria up front so you can verify your import later. In the final module (Evaluating your queries), you will run these same checks.

Use these scenarios to measure whether your import was meaningful:

* **Node counts** — The number of nodes per label matches the row counts in your source tables. If you have 91 customers in your relational source, you should have 91 `:Customer` nodes.
* **Relationship counts** — The number of relationships matches the expected connections (e.g. one `PLACED` per order, one `CONTAINS` per order line). No missing or duplicate edges.
* **Property integrity** — Key properties (IDs, names, dates) have the correct values and types when you sample nodes and relationships.
* **Referential integrity** — Every relationship connects two existing nodes; no orphans.
* **Sample queries** — At least one or two business questions you care about (e.g. "orders per customer", "products in an order") return correct results in Cypher compared with what you expect from the source.

Write down your source row counts and one or two example queries now. You will reuse them in the final module to validate your imported graph.


== Approaches for Importing Data

Data sources and import methods:

* Relational Database Management Systems (RDBMS)
* Web APIs
* Public data directories
* BI tools
* Excel
* Flat files (CSV, JSON, XML)

=== Import factors and options

Import method depends on these factors:

* The source of the data
* The volume of data
* The frequency of the import
* The complexity of the data model
* The transformation required

Import approaches (choose based on your data volume, update frequency, and transformation needs):

* One-off batch import of all data 
* One-off load with a regular update
* Continuous import of data
* Real-time application updates
* ETL (Extract, Transform, Load) pipelines
