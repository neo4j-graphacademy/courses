== Scenario

This minimal-effort challenge uses the testing methodology from Lesson 1. You will prepare the validation checklist you will use in the final module to confirm your import was meaningful—and complete one for Northwind as an example.


== Your task

=== 1. Prepare your own validation checklist (if you have source data)

Using the criteria from Lesson 1, write down:

* **Node counts** — For each table you plan to turn into a node label, record the current row count in your source (e.g. customers: _____, orders: _____, products: _____).
* **Relationship counts** — For each relationship type you expect (e.g. one per order, one per order line), note how many you expect or how you will derive it (e.g. "same as order count", "count of order_details rows").
* **One or two sample queries** — Write the business question and, if possible, the SQL or the expected result (e.g. "How many orders per customer?" or "List products in order X"). You will run the equivalent in Cypher in the final module.

Keep this checklist; you will use it when evaluating your imported graph.
