[.question]
= Tables That Do Not Become Nodes

Which of the following would typically NOT result in creating a node in the graph? Select all that apply.

* [x] A junction table with only two foreign key columns
* [ ] A table representing customers with contact information
* [x] A small lookup table with 3 static values like "High", "Medium", "Low"
* [ ] A table representing products with pricing and inventory data

[TIP,role=hint]
.Hint
====
Junction tables with only foreign keys become relationships. Small lookup tables often become properties or a single node type; core entity tables such as customers and products become nodes.
====

[TIP,role=solution]
.Solution
====
**Junction tables with only foreign keys** and **small lookup tables with static values** typically do not become nodes.

* **Junction tables** become relationships (e.g. `employee_territories` → `ASSIGNED_TO`).
* **Small lookup tables** (e.g. High/Medium/Low) can become a property on the node or a single reference node.

**Customers** and **products** are entity tables and become nodes because they are core business entities with meaningful properties.
====
