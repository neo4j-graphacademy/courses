== Course Description

In this workshop, you will get a conceptual map of **Microsoft Agent Framework (MAF)**: how it relates to Microsoft’s earlier agent stacks, how **agents**, **tools**, and **orchestration** fit together, and how **Model Context Protocol (MCP)** and **agent-to-agent (A2A)** patterns connect to the GraphRAG and Neo4j ideas from link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^] (and deeper knowledge-graph courses when you need them).

The session is **concept-first** (about 30–45 minutes) and does not replace hands-on Neo4j labs.

== Duration

{duration}

== What you will learn

* The purpose and scope of Microsoft Agent Framework
* How agents, tools, and orchestration fit together in MAF-shaped applications
* How Model Context Protocol (MCP) and agent-to-agent (A2A) patterns fit into integration and multi-agent setups
* How Neo4j, GraphRAG, and MCP-related patterns extend what you learned in link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^] and related GraphAcademy GenAI courses

== Prerequisites

**Complete link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^] on GraphAcademy before this workshop.** That online, self-paced course is the expected prerequisite. It covers embeddings, vector indexes, RAG, GraphRAG, and the Neo4j GraphRAG for Python package; this session assumes you already recognize those ideas.

If you need hands-on knowledge graph construction beyond that baseline, add one of:

* link:/courses/llm-knowledge-graph-construction/[Building Knowledge Graphs with LLMs^] - LLM-assisted graph building and querying
* link:/courses/genai-graphrag-python/[Constructing Knowledge Graphs with Neo4j GraphRAG for Python^] - `neo4j_graphrag` pipelines, retrievers, and customization
