== Introduction

**Microsoft Agent Framework** is an open, cross-language SDK for building **AI agents** and **multi-agent** systems on top of modern language models. Microsoft positions it as a single place to model agents, connect **tools**, run **orchestrated workflows**, and integrate with protocols such as **Model Context Protocol (MCP)**—the same protocol many Neo4j and GraphAcademy labs use for tool calling.

This lesson stays conceptual. After link:/courses/workshop-genai/[Neo4j and Generative AI Workshop^], you already know retrieval, embeddings, and conversational patterns. Here you add a **runtime and library lens**: how a mainstream agent framework packages those ideas for production-style applications.

[.slide]
== Why a dedicated framework?

Teams outgrow one-off scripts when they need **consistent** patterns for:

* **Tool use** (calling APIs, databases, or MCP servers with structured inputs)
* **Multi-step workflows** (branching, retries, human approval)
* **Multiple agents** (specialists that hand off work or run in parallel)
* **Observability** (streaming, logging, checkpoints)

A framework does not replace good application design. It gives you **shared vocabulary**, **reusable components**, and **documentation** so more than one developer can maintain the system.

[.slide]
== Lineage and scope

Microsoft Agent Framework builds on ideas from **Semantic Kernel** and **AutoGen**, combining orchestration and agent abstractions into a **unified** programming model for **Python** and **.NET**. The exact feature set evolves with each release; treat the official repository and release notes as the source of truth for API names and package layout.

Expect **multi-provider** model support (commercial and open weights), **graph-structured workflows** in higher-level samples, and emphasis on **interoperability** with MCP and agent-to-agent patterns—both relevant when Neo4j appears as a tool or knowledge source.

[.slide]
== How you work with MAF in practice

Typical steps (at a high level):

. Pick a **language runtime** (Python or .NET) and add the published packages from the official instructions.
. Define one or more **agents** with instructions and allowed **tools**.
. Run **single-turn** or **multi-turn** flows; scale up to **orchestrated** graphs when the task splits cleanly across steps or roles.

You still choose **where truth lives**—documents, a graph database, vector indexes—and how retrieval runs. MAF helps you **wire** models, tools, and control flow in code.
