== Introduction

The GenAI workshop already combined **graphs**, **vectors**, and **retrieval**. Microsoft Agent Framework does not replace Neo4j; it gives you a **structure** for calling Neo4j-backed logic as **tools** inside agent loops—whether that logic is **Cypher**, **GraphRAG** pipelines, or hybrid search.

Think in layers:

* **Data:** Neo4j holds entities, relationships, and optional vector indexes.
* **Retrieval:** GraphRAG packages (for example **neo4j-graphrag** patterns from the workshop) produce context bundles for the model.
* **Agent runtime:** MAF schedules the conversation, tool calls, and orchestration around that retrieval.

[.slide]
== Neo4j as the knowledge layer

In production sketches, Neo4j often plays one or more roles:

* **Source of truth** for curated facts and topology
* **Hybrid retrieval** (vector + graph expansion) for grounded answers
* **Schema** that **Text-to-Cypher** or templated queries can target

Expose those capabilities to the agent **through tools**—small, well-tested entry points—rather than letting the model invent connection strings or ad-hoc queries in unsafe settings.

[.slide]
== MCP and Neo4j tooling

When Neo4j (or your app) is reachable **through MCP**, the same tool surface can plug into **multiple** agent hosts—not only MAF. That aligns with how GraphAcademy teaches **MCP**: standardize the **tool boundary**, keep credentials and query policy **outside** the model weights.

If you have not walked through Neo4j’s MCP-oriented material yet, use link:/courses/genai-mcp-neo4j-tools/[Developing with Neo4j MCP Tools^] as a hands-on complement to this mini workshop.

[.slide]
== What to do next

. **Deepen GraphRAG:** Repeat or extend labs from link:/courses/workshop-genai/[Neo4j and Generative AI Workshop^]; compare retrieval quality with and without graph expansion.
. **Study MAF samples:** Work from link:https://github.com/microsoft/agent-framework[github.com/microsoft/agent-framework^] and the linked sample repositories; run the smallest Python or .NET agent that calls a tool you control.
. **Connect the two:** Implement a tool that wraps your Neo4j retrieval function, then call it from an MAF agent loop and measure latency, cost, and failure modes.

You now have a **roadmap**: Neo4j and GraphRAG for **grounding**, MAF for **agent orchestration**, MCP where you need **portable** tool interfaces.
