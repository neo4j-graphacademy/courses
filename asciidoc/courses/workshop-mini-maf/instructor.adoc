= Microsoft Agent Framework (MAF) Workshop — Facilitators guide

link:https://graphacademy.neo4j.com/courses/workshop-mini-maf/[graphacademy.neo4j.com/courses/workshop-mini-maf/]

== Introduction

This workshop is a **short conceptual add-on** (about 30–45 minutes of teaching time) for learners who have completed link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^] (or bring equivalent experience). It introduces **Microsoft Agent Framework (MAF)** in product-neutral terms: agents, tools, orchestration, **Model Context Protocol (MCP)**, and **agent-to-agent (A2A)** patterns—and sketches how that mental model connects to **Neo4j**, **GraphRAG**, and GraphAcademy MCP content.

There is **no** required coding in GraphAcademy for this course. If you extend the session with live demos, do that outside the course UI (for example a local Python or .NET sample from Microsoft’s repositories).

The session works **online or in person**. Keep discussion time explicit in your agenda; the content is dense for a single short block.

== Prerequisites (for learners)

Learners get the most value if they have already worked through retrieval, embeddings, and GraphRAG—for example by completing link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^].

Optional next depth (especially if your audience lacks hands-on KG construction):

* link:/courses/llm-knowledge-graph-construction/[Building Knowledge Graphs with LLMs^]
* link:/courses/genai-graphrag-python/[Constructing Knowledge Graphs with Neo4j GraphRAG for Python^]

Optional MCP reinforcement:

* link:/courses/genai-mcp-neo4j-tools/[Developing with Neo4j MCP Tools^] (helps when you discuss MCP in module 2–3)

== Learning objectives (what learners should leave with)

* A one-sentence description of what Microsoft Agent Framework is and why frameworks exist for agents.
* A clear split between **agent** (policy + model + tools) and **tool** (narrow adapters).
* A working mental model of **MCP** as a tool boundary—and how that links to Neo4j-related MCP material on GraphAcademy.
* A layered picture: **Neo4j / GraphRAG** as data and retrieval, **MAF** as one possible agent runtime, **MCP** where portability across hosts matters.

== Outline plan

All timings are estimates. Total **30–45 minutes** of content; add 10–15 minutes for Q&A if you can.

=== Module 1 — What is Microsoft Agent Framework? (≈ 10–12 minutes)

*Goals:* Framework vs scripts; lineage (Semantic Kernel, AutoGen); Python / .NET; “where truth lives” vs wiring.

*Facilitation tips:*

* Start from fundamentals-level GraphRAG: retrieval and tools are not new—**MAF names the runtime and patterns** around them.
* Emphasize **official repo + release notes** over memorized API details.
* Ask: “Where would you put **Neo4j** in this picture on a project you know?”

=== Module 2 — Agents, tools, workflows (≈ 12–15 minutes)

*Goals:* Agent vs tool; MCP; A2A and orchestration at a high level.

*Facilitation tips:*

* Draw a quick diagram: user → agent → tool → Neo4j (or MCP server) → back.
* Connect explicitly to link:/courses/genai-mcp-neo4j-tools/[Developing with Neo4j MCP Tools^] so learners see **one** MCP story across GraphAcademy.
* Ask: “Name one **bad** tool interface for an LLM (too big, no types) and one **good** one.”

=== Module 3 — Neo4j, GraphRAG, next steps (≈ 10–12 minutes)

*Goals:* Layers: data, retrieval, agent runtime; MCP boundary; concrete next steps.

*Facilitation tips:*

* Repeat the three-layer list (data / retrieval / agent runtime) slowly; learners will map their own systems differently.
* Point to **follow-on courses** in the lesson and in `summary.adoc` instead of inventing new links in the room.
* Ask: “What would you **tool-ify** first from your GraphRAG pipeline?”

=== Close (≈ 5–10 minutes)

* Recap the four learning objectives in your own words.
* Run Q&A.
* Remind learners that GraphAcademy enrollment **stays available** for review.
* Suggest next steps: deeper GraphRAG, MCP courses, MAF samples on GitHub.

== Discussion prompts (optional)

* When does a **framework** earn its keep versus adding **moving parts** you do not need yet?
* How is **MCP** different from “just REST” for your team’s security and deployment story?
* Where would **human-in-the-loop** appear in a graph-backed agent you would ship?

== Common pitfalls

* **Version drift:** Do not read package names from memory—confirm against Microsoft’s repo before you demo.
* **Conflating MAF and Neo4j:** Neo4j is not “inside” MAF; MAF is one possible **host** for tools that talk to Neo4j.
* **Over-promising:** This course does not certify production readiness for MAF; it orients learners for self-study and pilots.
