== Course Description

AI agents share common problems: hallucinations without real data, lack of domain context, lack of memory across turns, and unreliable retrieval when the agent must choose to call a tool.

A **self-grounding agent** addresses these by automatically receiving relevant knowledge from a graph database on every interaction. The Microsoft Agents Framework supports this through **context providers**, which run before and after every LLM call to inject knowledge and extract state.

This workshop teaches you to build self-grounding agents using MAF and Neo4j: vector, fulltext, and hybrid retrieval; graph-enriched search; and persistent memory with entity extraction and optional graph algorithms.

This course uses models from link:https://openai.com/[OpenAI^], although you can use the model and supplier of your choice.

To complete the practical tasks, you will need an link:https://platform.openai.com[OpenAI API key^]. If you are attending the **Road to NODES AI** workshop, an OpenAI API key may be provided for you.

Familiarity with Neo4j and basic Python helps.

== What you will learn

* What self-grounding agents are and how context providers enable them
* How the Microsoft Agent Framework differs from other agent frameworks
* The context provider lifecycle (`before_run` and `after_run`) and how it differs from tools
* How to build custom context providers that inject dynamic instructions
* How to use `Neo4jContextProvider` with vector, fulltext, and hybrid search modes
* How to add graph-enriched retrieval using custom Cypher traversal queries
* How to give agents persistent memory with entity extraction, reasoning traces, and graph algorithms
