= Neo4j Agent Memory Context Provider

The context provider from Module 3 injected pre-existing knowledge graph data into agent conversations. That data was static, loaded into Neo4j before the agent ever ran.

The link:https://github.com/neo4j-labs/agent-memory[neo4j-agent-memory^] package takes a different approach. Rather than retrieving existing knowledge, it captures what the agent learns _during_ conversations — user preferences, entity mentions, and reasoning traces — then stores them in Neo4j for retrieval in future turns and sessions. This gives agents persistent memory that grows over time.

Unlike `agent-framework-neo4j`, which delegates search to a separate retrieval library (`neo4j-graphrag`), `neo4j-agent-memory` has the context provider built in. The package handles both the MAF integration and the Neo4j storage directly, providing `Neo4jMicrosoftMemory` as a unified interface for memory operations. It hooks into the same `before_run()` and `after_run()` lifecycle methods — retrieving relevant memories before each LLM call and extracting new information from the conversation afterward.

The package provides three memory types (short-term, long-term, and reasoning), a multi-stage entity extraction pipeline, and optional graph algorithm integration. It integrates with MAF through both context providers and callable tools.

In this module, you will:

* Understand the three memory types: short-term, long-term, and reasoning
* Learn how the multi-stage entity extraction pipeline works
* Use `Neo4jMicrosoftMemory` as a context provider for automatic memory injection
* Combine passive memory (context providers) with active memory (memory tools)
* Use reasoning memory to learn from past agent executions
* Optionally integrate graph algorithms for advanced entity analysis
