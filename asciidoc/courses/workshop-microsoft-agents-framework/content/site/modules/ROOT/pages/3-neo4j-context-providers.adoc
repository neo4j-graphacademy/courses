= Neo4j Knowledge Retrieval Context Provider

In the previous module, you learned how context providers work in the Microsoft Agent Framework and built one from scratch. Your `UserInfoMemory` provider injected instructions and extracted structured data, but the knowledge it worked with was limited to what appeared in the conversation itself.

Most real-world agents need access to external knowledge — product catalogs, support articles, research data, or in this case, a movie database. Now you will use `Neo4jContextProvider` to give your agents automatic access to knowledge stored in Neo4j, grounding every response in real data without requiring tool calls.

Neo4j provides two context provider packages for MAF, each solving a different problem:

* **link:https://github.com/neo4j-labs/neo4j-maf-provider[agent-framework-neo4j^]** retrieves knowledge from Neo4j indexes (vector, fulltext, or hybrid search) and injects it before every LLM call. Use this to ground agents in your existing knowledge graph data.
* **link:https://github.com/neo4j-labs/agent-memory[neo4j-agent-memory^]** gives agents persistent memory across conversations. Captures user preferences, entity mentions, and reasoning traces during conversations and stores them in Neo4j for future retrieval.

This module focuses on `agent-framework-neo4j`. Its `Neo4jContextProvider` hooks into the same `before_run()` lifecycle method you used in Module 2, but instead of injecting hardcoded instructions, it searches a Neo4j index and injects the matching results as context. The agent never requests this information explicitly; retrieval happens transparently on every turn. The next module covers `neo4j-agent-memory` for persistent agent memory.

In this module, you will:

* Understand the `agent-framework-neo4j` package and how `Neo4jContextProvider` extends `BaseContextProvider`
* Use vector search for semantic similarity over movie plot embeddings
* Add graph-enriched retrieval using custom Cypher traversal queries
* _(Optional)_ Use fulltext search for keyword matching without an embedder
* _(Optional)_ Combine both with hybrid search for the best of both approaches
