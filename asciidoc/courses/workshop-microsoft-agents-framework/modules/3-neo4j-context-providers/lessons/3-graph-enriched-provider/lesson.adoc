= Graph-Enriched Retrieval
:order: 3
:type: lesson

TIP: Complete this lesson hands-on by opening `labs/lab-4-graph-enriched.ipynb` in your Codespace.

In the previous lesson, you used vector search to find relevant movies by semantic similarity. The search returned matching plot text, but your graph contains far more than text. Each movie is connected to genres, actors, and directors through typed relationships. Basic search misses all of that structure.

Graph-enriched retrieval adds a Cypher traversal step after the index search. The provider finds matching movies, then walks the graph to collect genres, cast, and directors for each match. The agent receives both the relevant text and the structured metadata.

== The Limitation of Basic Search

The vector search from the previous lesson returns matching text and a similarity score for each result. If you search for "time travel," you get plots that mention time travel concepts. But the result doesn't include the movie's title, release year, genres, or who starred in it.

That metadata lives in the graph: `Movie` nodes connect to `Genre` nodes through `IN_GENRE`, to actors through `ACTED_IN`, and to directors through `DIRECTED`. Graph enrichment traverses these relationships after the initial search.

== Define a Retrieval Query

The `retrieval_query` parameter accepts a Cypher query that runs after the index search. It receives two variables:

* **`node`** -- The matched `Movie` node from the index
* **`score`** -- The relevance score from the search

The query traverses outward from the matched node:

[source,cypher]
----
// Receives :param node and :param score from the index search
MATCH (node)-[:IN_GENRE]->(g:Genre)
WITH node, score, collect(DISTINCT g.name) AS genres
OPTIONAL MATCH (p:Person)-[:ACTED_IN]->(node)
WITH node, score, genres, collect(DISTINCT p.name)[0..5] AS actors
OPTIONAL MATCH (d:Person)-[:DIRECTED]->(node)
WITH node, score, genres, actors, collect(DISTINCT d.name) AS directors
WHERE score IS NOT NULL
RETURN
    node.plot AS text,
    score,
    node.title AS title,
    node.released AS released,
    genres,
    actors,
    directors
ORDER BY score DESC
----

This query starts from each matched movie and collects:

* Genres: `Movie` -> `IN_GENRE` -> `Genre`
* Up to 5 actors: `Person` -> `ACTED_IN` -> `Movie`
* Directors: `Person` -> `DIRECTED` -> `Movie`

The result includes plot text, similarity score, movie title, release year, genre names, actor names, and director names. Far richer than plot text alone.

== Create the Graph-Enriched Provider

When you provide a `retrieval_query`, the provider internally uses a Cypher-enhanced retriever (e.g., `VectorCypherRetriever` for vector mode, `HybridCypherRetriever` for hybrid mode) instead of the basic retriever. This example uses vector search:

Define the retrieval query as a Python string and pass it to the provider using the `retrieval_query` parameter:

[source,python]
----
# Create context provider with graph-enriched retrieval
provider = Neo4jContextProvider(
    uri=neo4j_settings.uri,
    username=neo4j_settings.username,
    password=neo4j_settings.get_password(),
    index_name=neo4j_settings.vector_index_name,

----

The only difference from the vector search lesson is the `retrieval_query` parameter. Everything else stays the same.

== Run the Agent

[source,python]
----
async with provider:
    client = get_client()

    agent = client.as_agent(
        name="graph-enriched-agent",
        instructions=(
            "You are a helpful assistant that answers questions about "
            "movies using graph-enriched context. Your context includes:\n"
            "- Semantic search matches from movie plots\n"
            "- Movie titles and release years\n"
            "- Genres the movie belongs to\n"
            "- Actors who appeared in the movie\n"
            "- Directors of the movie\n\n"
            "When answering, cite the movie title, relevant actors, and genres. "
            "Be specific and reference the enriched graph data."
        ),
        context_providers=[provider],
    )

    session = agent.create_session()

    query = "What are some good science fiction movies and who stars in them?"
    response = await agent.run(query, session=session)
    print(response.text)
----

The agent now receives context that includes plot text, movie title, release year, genres, actors, and directors for each match. It can give specific answers: "The Matrix (1999), directed by Lana and Lilly Wachowski, is a sci-fi action film about..." instead of a vague summary based on training data.

== Experiment

Try different retrieval query patterns:

* Add `node.imdbRating` to the RETURN clause to include ratings
* Change the actor limit from 5 to 3 or 10 and see how it affects context size
* Modify `top_k` to retrieve more or fewer matches
* Update the `context_prompt` to tell the agent to prioritize higher-rated movies

import asyncio
import os

from dotenv import load_dotenv
load_dotenv()

from agent_framework.openai import OpenAIResponsesClient
from agent_framework_neo4j import Neo4jContextProvider, Neo4jSettings
from neo4j_graphrag.embeddings.openai import OpenAIEmbeddings

# Load settings from environment variables
neo4j_settings = Neo4jSettings()

# Create embedder
embedder = OpenAIEmbeddings(model="text-embedding-3-small")

# Graph-enriched retrieval query
# Appended after vector search: YIELD node, score
RETRIEVAL_QUERY = """
MATCH (node)-[:IN_GENRE]->(g:Genre)
WITH node, score, collect(DISTINCT g.name) AS genres
OPTIONAL MATCH (p:Person)-[:ACTED_IN]->(node)
WITH node, score, genres, collect(DISTINCT p.name)[0..5] AS actors
OPTIONAL MATCH (d:Person)-[:DIRECTED]->(node)
WITH node, score, genres, actors, collect(DISTINCT d.name) AS directors
WHERE score IS NOT NULL
RETURN
    node.plot AS text,
    score,
    node.title AS title,
    node.released AS released,
    genres,
    actors,
    directors
ORDER BY score DESC
"""

# Create context provider with graph-enriched retrieval
provider = Neo4jContextProvider(
    uri=neo4j_settings.uri,
    username=neo4j_settings.username,
    password=neo4j_settings.get_password(),
    index_name=neo4j_settings.vector_index_name,
    index_type="vector",
    retrieval_query=RETRIEVAL_QUERY,
    embedder=embedder,
    top_k=5,
    context_prompt=(
        "## Graph-Enriched Movie Context\n"
        "The following information combines semantic search results with "
        "graph traversal to provide movie, actor, genre, and director context:"
    ),
)

async def main():
    async with provider:
        client = OpenAIResponsesClient()

        agent = client.as_agent(
            name="graph-enriched-agent",
            instructions=(
                "You are a helpful assistant that answers questions about "
                "movies using graph-enriched context. Your context includes:\n"
                "- Semantic search matches from movie plots\n"
                "- Movie titles and release years\n"
                "- Genres the movie belongs to\n"
                "- Actors who appeared in the movie\n"
                "- Directors of the movie\n\n"
                "When answering, cite the movie title, relevant actors, and genres. "
                "Be specific and reference the enriched graph data."
            ),
            context_providers=[provider],
        )

        session = agent.create_session()

        query = "What are some good science fiction movies and who stars in them?"
        print(f"User: {query}\n")
        print("Answer: ", end="", flush=True)
        response = await agent.run(query, session=session)
        print(response.text)
        print()

asyncio.run(main())

== Lesson Summary

Adding a `retrieval_query` to `Neo4jContextProvider` combines any search mode with graph traversal. The Cypher query receives matched `node` and `score` variables from the index search, then traverses relationships to collect genres, actors, and directors. The agent gets structured metadata alongside plot text, producing more specific and grounded responses.
