Vector search understands concepts but can miss exact terms. Fulltext search matches keywords precisely but misses semantic connections. Hybrid search runs both and combines the results, giving you semantic understanding and keyword precision in one query.

== How Hybrid Search Works

When `index_type` is set to `"hybrid"`, the provider:

1. Embeds the query and runs a vector search against the vector index
2. Tokenizes the query and runs a fulltext search against a separate fulltext index
3. Combines the scores from both searches to produce a single ranked result set

This means a movie can rank highly because its plot is semantically similar to the query, because it contains the exact keywords, or both. Results that score well on both dimensions rise to the top.

== Requirements

Hybrid search requires:

* A **vector index** with embeddings on the target nodes (the `index_name` parameter)
* A **fulltext index** on the same or related text (the `fulltext_index_name` parameter)
* An **embedder** to generate query vectors

Both indexes must exist in Neo4j before the provider can use them.

== Configure the Provider

neo4j_settings = Neo4jSettings()

----

[source,python]
----
# Create context provider with hybrid search
provider = Neo4jContextProvider(
    uri=neo4j_settings.uri,
    username=neo4j_settings.username,
    password=neo4j_settings.get_password(),
    index_name=neo4j_settings.vector_index_name,
    index_type="hybrid",
    fulltext_index_name=neo4j_settings.fulltext_index_name,
    embedder=embedder,
    top_k=5,
    context_prompt=(
        "## Movie Knowledge Graph Context\n"
        "Use the following movie information from the knowledge graph "
        "to answer questions:"
    ),
)

----

The key difference from vector-only search is the `fulltext_index_name` parameter, which tells the provider which fulltext index to use alongside the vector index.

== Run the Agent

[source,python]
----
async def main():
    async with provider:
        client = OpenAIResponsesClient()

        agent = client.as_agent(
            name="movie-hybrid-agent",
            instructions=(
                "You are a movie recommendation assistant. Answer questions "
                "using the movie data provided in your context."
            ),
            context_providers=[provider],
        )

        session = agent.create_session()

        query = "Find movies about dream-sharing technology"
        print(f"User: {query}\n")
        print("Answer: ", end="", flush=True)
        response = await agent.run(query, session=session)
        print(response.text)
        print()

asyncio.run(main())

----

This query benefits from both search strategies:

* **Fulltext search** finds plots containing "dream-sharing" and "technology" as literal terms, matching Inception's plot directly
* **Vector search** finds movies semantically related to dream and technology concepts, even if those exact words don't appear in the plot

Hybrid combines both result sets. A movie like Inception ranks highly because it matches on both dimensions.

== When to Use Hybrid

Hybrid search is the most versatile mode but comes with trade-offs:

* **Advantages**: Catches both semantic and keyword matches, reduces the chance of missing relevant results
* **Trade-offs**: Requires both indexes to exist, needs an embedder (adds latency), uses more resources than either mode alone

Use hybrid when your users mix conceptual and keyword-based queries. Use vector-only when queries are primarily conceptual. Use fulltext-only when queries are primarily keyword-based and you want the simplest setup.

== Graph Enrichment

Like the other search modes, hybrid search supports the `retrieval_query` parameter for graph enrichment. When provided, the provider uses `HybridCypherRetriever` instead of the basic `HybridRetriever`. The retrieval query works the same way as with vector search: it receives `node` and `score` from the combined search results and can traverse relationships to collect additional context.

Graph enrichment is covered in detail in the Graph-Enriched Retrieval lesson earlier in this module.

import asyncio
import os

from dotenv import load_dotenv
load_dotenv()

from agent_framework.openai import OpenAIResponsesClient
from agent_framework_neo4j import Neo4jContextProvider, Neo4jSettings
from neo4j_graphrag.embeddings.openai import OpenAIEmbeddings

# Load settings from environment variables
neo4j_settings = Neo4jSettings()

# Create embedder for converting queries to vectors
embedder = OpenAIEmbeddings(model="text-embedding-3-small")

# Create context provider with hybrid search
provider = Neo4jContextProvider(
    uri=neo4j_settings.uri,
    username=neo4j_settings.username,
    password=neo4j_settings.get_password(),
    index_name=neo4j_settings.vector_index_name,
    index_type="hybrid",
    fulltext_index_name=neo4j_settings.fulltext_index_name,
    embedder=embedder,
    top_k=5,
    context_prompt=(
        "## Movie Knowledge Graph Context\n"
        "Use the following movie information from the knowledge graph "
        "to answer questions:"
    ),
)

async def main():
    async with provider:
        client = OpenAIResponsesClient()

        agent = client.as_agent(
            name="movie-hybrid-agent",
            instructions=(
                "You are a movie recommendation assistant. Answer questions "
                "using the movie data provided in your context."
            ),
            context_providers=[provider],
        )

        session = agent.create_session()

        query = "Find movies about dream-sharing technology"
        print(f"User: {query}\n")
        print("Answer: ", end="", flush=True)
        response = await agent.run(query, session=session)
        print(response.text)
        print()

asyncio.run(main())
