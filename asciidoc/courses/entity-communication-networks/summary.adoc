= Course summary
:pdf-summary:

Congratulations on completing **Entity Communication Networks**!

You've gone from 4,911 raw documents to a live communication network in Neo4j -- extracting text, handling OCR noise, training an NER model, and building a hybrid pipeline that combines templates, NER, and LLM parsing.

== What you built

* Extracted text from PDFs using a per-page strategy: PyMuPDF for text layers, built-in OCR for image-only pages
* Investigated the corpus structure: boilerplate, header fields, body text, embedded chains, OCR noise
* Built regex templates that match exact header layouts -- and reject anything that doesn't fit
* Trained a spaCy NER model to identify individual people and email addresses within header fields
* Used an LLM to parse the emails that templates couldn't handle
* Combined all three into a hybrid pipeline: templates for structure, NER for entity splitting, LLM for edge cases
* Normalized raw values with unicode decomposition, artifact stripping, and an OCR domain correction table
* Imported the results into Neo4j Aura Free with constraints, MERGE on normalized values, and raw values preserved as properties

== What's in your graph

* **Email** nodes with subject, date, body, and parse method
* **User** nodes for every named sender and recipient
* **Mailbox** nodes for every email address
* **Domain** nodes extracted from email addresses
* **SENT**, **RECEIVED**, **CC_ON**, **USED**, and **HAS_MAILBOX** relationships connecting them

== Key takeaways

* Investigate your data before writing parsing code -- what you find shapes every decision
* The per-page extraction strategy (text layer if available, OCR if not) handles mixed PDFs without heuristics
* Re-OCR on clean images recovers what older OCR passes got wrong
* Templates are rejection tools -- they match exactly or fail cleanly, never silently producing wrong data
* NER identifies individual entities within raw field strings, handling `Last, First` names and mixed formats
* LLMs earn their cost only on inputs that other methods can't handle
* Normalize for merging, but preserve raw values -- you can always trace back to the source

== The prompt pack

The prompt pack is a project folder with LLM instructions that applies the techniques from this course to your own data. Open it in Claude Code, Cursor, or your IDE of choice, point it at your documents, and it builds a pipeline tailored to your corpus.
