== Introduction

Microsoft Agent Framework does not replace Neo4j. 

MAF gives you a **structure** for calling Neo4j-backed logic as **tools** from agents. The tools could be **Cypher** queries, **GraphRAG** pipelines, or hybrid search. 

You should structure the system in three layers:

* **Data:** Neo4j holds entities, relationships, and optional vector indexes.
* **Retrieval:** GraphRAG packages (for example **neo4j-graphrag** patterns from link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^] ) produce context bundles for the model.
* **Agent runtime:** MAF schedules the conversation, tool calls, and orchestration around that retrieval.

[.slide]
== Neo4j as the knowledge layer

In typical architectures, Neo4j often plays one or more roles:

* **Source of truth** for curated facts and topology
* **Hybrid retrieval** (vector + graph expansion) for grounded answers
* **Schema** that **Text-to-Cypher** or templated queries can target

Expose those capabilities to the agent **through tools** - small, well-tested entry points - rather than letting the model invent connection strings or ad-hoc queries in unsafe settings.

[.slide]
== MCP and Neo4j tooling

When Neo4j (or your app) is reachable **through MCP**, the same tool surface can plug into **multiple** agent hosts - not only MAF. **MCP** standardizes the **tool boundary**, keeps credentials and query policy **outside** the model weights.

If you have not walked through Neo4j’s MCP-oriented material yet, use link:/courses/genai-mcp-neo4j-tools/[Developing with Neo4j MCP Tools^] as a hands-on complement to this workshop.

[.slide]
== What to do next

. **Deepen GraphRAG:** Revisit link:/courses/genai-fundamentals/[Neo4j & Generative AI Fundamentals^] or go deeper with link:/courses/genai-graphrag-python/[Constructing Knowledge Graphs with Neo4j GraphRAG for Python^] and link:/courses/llm-knowledge-graph-construction/[Building Knowledge Graphs with LLMs^]; compare retrieval quality with and without graph expansion.
. **Study MAF samples:** Work from link:https://github.com/microsoft/agent-framework[github.com/microsoft/agent-framework^] and the linked sample repositories; run the smallest Python or .NET agent that calls a tool you control.
. **Connect the two:** Wire the official Neo4j MCP server into an MAF agent loop (see link:/courses/genai-mcp-neo4j-tools/[Developing with Neo4j MCP Tools^]); optionally register an additional tool for GraphRAG retrieval you implemented in link:/courses/genai-graphrag-python/[Constructing Knowledge Graphs with Neo4j GraphRAG for Python^] (or similar) when MCP’s built-in tools are not enough; measure latency, cost, and failure modes.
. **Explore agent memory:** When you need durable, queryable sessions and traces - not only retrieved facts - work through link:/courses/genai-context-graphs/[Context Graphs: Agent Memory with Neo4j^] and the `neo4j-agent-memory` library alongside your MAF tool wiring.

You now have a **roadmap**: Neo4j and GraphRAG for **grounding**, MAF for **agent orchestration**, MCP for **portable** tool interfaces, and graph-backed **memory** when agent state must outlive a single chat window.
